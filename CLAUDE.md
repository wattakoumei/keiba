# 競馬予想ハーネス (keiba)

「◯◯レースを分析して」と言うだけで、観点別の並列web調査 → 二層モデルでの統合を行い、
**勝率表・妙味・単複/三連系の買い目推奨**を出力するための Claude Code ハーネス。
レース結果が出た後に予測精度を検証し、観点・重みを改善するループを回せる。

## モデル哲学（二層モデル）

### 1. 能力レイヤー（その馬がどれだけ走れるか）

```
発揮能力 = 潜在能力 × 適性 × 状態 − 割引 (+偶然)
```

- **潜在能力**: その馬が本来持つ地力（能力指数・近走内容・血統）
- **適性**: このレース条件への噛み合い（コース/距離/馬場、展開/脚質/枠）
- **状態**: 当日に向けた仕上がり（調教、ローテ/馬体、当日気配）
- **割引**: 減点要因（出遅れ癖、気性難、脚部不安、斤量、不利枠 等）
- **偶然**: 点推定を過信しないためのブレ幅（確信度・頭数・展開不確実性から信頼区間として注記）

発揮能力を場全体で正規化して **各馬の「自分の勝率」%** を得る。

### 2. 市場レイヤー（買うか／買い目をどう決めるか）

```
妙味 = 自分の勝率 − 市場の勝率
期待値 EV = 自分の勝率 × オッズ
```

- 市場勝率は単勝オッズの逆数を控除率で正規化（overround 除去）して求める。
- 妙味がプラス（=市場が過小評価）かつ EV が閾値超の馬を買い目候補にする。
- 対象馬券: **単勝・複勝**（コア）、**三連複・三連単**（連対/複勝確率の近似で拡張）。

## 使い方

| コマンド | 役割 |
|----------|------|
| `/analyze-race <レース名 or URL or 出走表貼り付け>` | 観点別並列調査 → 勝率表・妙味・買い目を出力 |
| `/review-prediction <レース名> <結果>` | 予測 vs 実結果を検証し、重み改善案を提示 |

データは web 調査が主体。出走表・オッズなど崩れやすい核データは、取得が不確かなとき
ユーザーに URL 指定または貼り付けでの補強を依頼する。

## 用語

- **観点 (observation point / factor)**: 調査の最小単位。1観点 = 1サブエージェントが担当。
  カタログは `.claude/skills/analyze-race/references/observation-points.md`。
- **発揮能力スコア**: 観点を集約した最終的な走力推定。
- **自分の勝率**: 発揮能力を場全体で正規化した勝率%。
- **市場勝率**: オッズから逆算した市場の織り込み勝率%。
- **妙味 (edge)**: 自分の勝率 − 市場勝率。
- **EV (期待値)**: 自分の勝率 × オッズ。1.0 が損益分岐。

## データの場所

```
data/
  races/<race-id>/
    出走表.md            # 核データ（レース条件・出走馬・単複オッズ）
    research-<観点>.md   # 観点別サブエージェントの調査結果
    report.md            # 最終レポート
  predictions.jsonl      # 予測ログ（race, 馬, 自分勝率, 市場勝率, 妙味, 買い目, 賭け金）
  results.jsonl          # 実結果（着順）
```

`race-id` は `YYYYMMDD-開催-レース番号`（例: `20260531-tokyo-11`）を推奨。

## 改善ループの回し方

1. `/analyze-race` で予測 → `predictions.jsonl` に追記。
2. レース確定後、`/review-prediction` で `results.jsonl` に結果記録。
3. 予測と結果を突合し、キャリブレーション（勝率ビン別的中率・Brier スコア・ROI）を算出。
4. どの観点が系統的に過大/過小評価かを診断し、`scoring-model.md` の重み・閾値の修正案を提示。
5. ユーザー承認で重みを更新（バージョンを上げる）→ これが 1 周。
