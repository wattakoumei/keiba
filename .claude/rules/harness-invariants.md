# keiba ハーネス不変則（正本）

> このファイルは `.claude/rules/` 内なので**セッション起動時に自動ロード**される。
> ハーネスの**強制ルールの正本**＝CLAUDE.md や各 reference の方針ブロックと食い違ったら**ここが勝つ**。
> 不変則を変えるときは**まずここを直し**、各文書のエコーを追従させる（手順は [editing-map](editing-map.md)）。

レポート（`/analyze-race`）・ログ・採点（`/review-prediction`）すべてに常時かかる。番号は優先度でなく参照用。

## I1. 市場ゼロ（純粋情報の原則）＝**予想ハーネス（`/analyze-race`）に適用**
- オッズ・人気・他人の予想（専門紙の印・予想展開記事）は**証拠にもログにも一切使わない**。
- **買い目・EV・Kelly・妙味・期待値・回収率は出さない**。馬券選択・実ベットは**人間判断**。
- 予測は馬の内在情報のみで完結させる。
- 適用範囲は**予想本体**＝`/analyze-race` とその成果物（report.json／predictions.jsonl／research-*.json）。**選別レイヤーは I1-S の隔離条件下でのみ市場を使う**。

## I1-S. 選別レイヤーの例外（市場の隔離・`/screen-card`）
- 勝負レース選別ハーネス（`/screen-card`）は**予想とは別レイヤー**として、レース選別の目的に限り**市場（単勝オッズ）を読んでよい**（団子度＝Y軸の当日側）。
- **隔離の壁（破ってはならない）**:
  1. オッズ取得（`tools/fetch_odds.py`）の出力は **`data/screening/` にのみ**書く。`research-*.json`／`report.json`／`predictions.jsonl` には**一切流さない**。
  2. `/analyze-race`（予想本体）は従来どおりオッズを**一切見ない**＝X軸（誰が勝つか）は内在情報だけで作る。選別の団子度は X の証拠にしない。
  3. 選別の出力は**レース単位の妙味判定まで**（🟥本命勝負／🟦鉄板コツコツ／見送り）。**馬ごとの買い目・金額・EV・賭け金は出さない**（I1 の「買い目は人間」は選別でも維持）。
- 思想: 市場は「賭ける瞬間だけの隔離レイヤー」。純度は予想側で守られる。

## I2. %禁止（論理ファースト）＝**予想レポート（`/analyze-race`）に適用**
- レポートに **% を一切出さない**（発揮能力／1着率／勝率レンジ／複勝率／期待着順／条件付き% を含め全廃）。
- 強弱は **印 ◎◯▲△× と行順**で表す。確からしさは**可能性ティア 本線★★★／対抗★★／伏線★**（%でない）。
- 例外: **選別レイヤー（`/screen-card`）は事実ベースの % を出してよい**（オッズ・荒れ条件の基準率＝万馬券率/人気別勝率など）。これは予測した確率でなく**実測の市場/統計値**＝偽の精度ではないため。予想本体の「予測%禁止」とは別物。

## I3. 表中心・最小分量（ユーザーは表しか見ない）
- **作らないもの**: §1 サマリ（「当日の分岐点」1行のみ可）／**mermaid 等の図**／観点別ハイライト。
- 段階フロー（phase_flow）は**1行テキストの因果文**で書く（図にしない）。
- 数字の代わりに **展開列・展開感度・好材料・懸念点** を厚く。各根拠は **[観点タグ A〜K,I] ＋事実＋なぜそう読んだか**。
- 重複を出さない＝読む量を増やさない（結論の正本は §3 着順予想表。他所で繰り返さない）。

## I4. 2成果物の独立
- **成果物1＝展開予想（§2）** と **成果物2＝着順予想（§3）** を**別個に測り・別個に改善**する。
- 展開→着順の**因果は維持**（最有力パターンの phase_flow に各馬を通して並びを論理判断）。
- 各成果物は冒頭に**検証契約**を持ち、レース後に独立採点（展開精度／着順精度の2スコアカード）。

## I5. 展開は複数パターン＋当日可変
- 単一に潰さない。**名前付きパターン**（可能性ティア＋発動トリガー＋pace_level＋脚質別有利不利＋隊列＋phase_flow）で持つ。
- 当日情報（参考R・馬場・乗替・取消）で**ティアを付け替え／1パターンに固定 → 展開感度・並び・展開列を論理で再評価**（素の能力読み＝好材料/懸念点は再調査不要）。

## I6. 確定材料の先取り（§0 後付けにしない）
- 枠順・乗替・回避（取消）が分析時点で確定済みなら、**§2-1/§2-2/§3 本文に最初から織り込む**（脚質への内/外割当て・先行争い再評価まで）。
- §0 当日ボードに残すのは*分析時点で本当に未知のものだけ*（当日馬場・クッション値・含水率・パドック・馬体重・前半参考Rの観察値）。
- 目的: 完成レポートに後付けして**ハーネスを再実行する無駄を避ける**。

## I7. 展開列（箱組みの効率化・成果物2の一部）
- §3 で各馬が**各パターンで圏内になるか・強弱**を一目化（`◎`中心／`○`圏内／`△`条件付き、**圏外は記載なし**。例 `全◎`／`β◎`／`α◎γ◎δ◎ β△`）。
- 源は展開合成の `per_horse_fit`・`leg_advantage`・各パターンの浮上/沈む馬。`predictions.jsonl` の `pattern_fit` にも残す。
- **§3 の直前に転置版「パターン別おすすめ馬」表**（パターン → 中心◎/圏内○/一発△/消し）を置く＝箱組みの逆引き。源は展開列と同一（`per_horse_fit`）で**矛盾させない**（重複でなく転置ビュー＝I3 の例外として許可）。
- 目的は「想定パターン別に無駄なく箱(ボックス)を組む」効率化のみ。**券種・金額・実ベットは人間**（I1 を破らない）。

## I8. エンジンは任意のサニティチェック
- `tools/score_race.py` は**並びの内部整合を確かめる任意のチェック**に降格。**% の正本ではない**。
- 出力の % は**レポートに転記しない**。論理の並びと食い違ったら**論理側を正**とし、食い違いを点検シグナルとして残す（`engine_check`）。

## I9. レポート構造（固定番号）
`§0 当日ボード` → `§1 当日の分岐点(1行)` → `§2 展開予想【成果物1】` → `§3 着順予想表【成果物2】` → `§4 データの確かさ` → `§5 免責`。
- **§2/§3 の番号は他文書（SKILL/各 reference）から参照されるので変えない**。体裁の正本は `skills/analyze-race/references/output-template.md`。

## I10. 静的データのワンソース化（再調査禁止）
- **不変の事実は正本から一度だけ供給し、web で再調査しない**: コース物理形状（直線・坂・初角・芝スタート）＝`references/course-geometry.md`、種牡馬の産駒傾向＝`references/pedigree-catalog.md`（半静的・年1更新／カタログ外の血だけ web 調査）、出走表スパイン・脚質/テン速・h2h直接対戦＝`tools/fetch_racecard.py` の seed。
- 供給経路は **spawn 時注入**（SKILL STEP1/3）。subagent は rules を自動ロードしない＝徹底は spawn データと各 agent 定義の記述が担う（誰が何に使うかは各 agent / pace-synthesis 側に書く）。
- 静的データの訂正は**正本ファイルだけ**直す（エコーを作らない）。web で採ってよいのは変動情報（当該開催の馬場・関係者コメント等）のみ。
