# calibrate-T — softmax温度の較正

`tools/score_race.py` の温度パラメータ T を、過去の予測結果から最適値に調整する。

## いつ使うか

- `/review-prediction` で結果が **5件以上** たまったとき
- 「確率が高すぎる/低すぎる」という定性的な違和感があるとき
- 20〜30レースごとの定期点検

## 手順

### STEP 1. 較正スクリプトを実行

```bash
python3 tools/calibrate_T.py
```

出力される較正レポート:
- T を 0.25〜1.50 の範囲で走査
- 各Tでの Brier Score（予測確率と実的中率のズレ）
- 現行Tと最適Tの比較
- 判定（維持/変更推奨/様子見/要検討）

### STEP 2. 判定に基づき対応

| 判定 | 対応 |
|------|------|
| **維持** | 現行Tと最適Tの差が小さい。何もしない |
| **変更推奨** | ユーザーに較正表を見せ、承認を得てから変更 |
| **様子見** | データ不足（5件未満）。レースを重ねてから再実行 |
| **要検討** | 最適Tでもskill<0＝T以外に問題がある（観点スコアの質など） |

### STEP 3. 変更を適用（ユーザー承認後）

```bash
python3 tools/calibrate_T.py --apply
```

これで `score_race.py` の T が更新される。**以下は手動で行う:**

1. `scoring-model.md` のパラメータ早見表を同値にミラー
2. 変更履歴にバージョンと理由を追記
3. 既存の report.json に `inject_probs.py` を再実行して率を更新

### STEP 4. 較正表をユーザーに提示

較正レポートの表をそのまま見せる。判断材料:
- **Brier Skill**: 正なら均等予測（全馬1/N）より良い。負なら予測が有害
- **◎単勝/◎複勝**: ◎の予測確率と実際の的中率が近いか
- **レース数**: 5件未満は統計的に不安定

## 注意

- T は「確率の鋭さ」だけを変える。**どの馬が強いかの順位は変わらない**
- 他の6ノブ（w_pos1 等）は着順の順位自体に影響する＝別の判断基準で調整（`/review-prediction` の A/B 仕分け）
- T の変更は `scoring-model.md` の変更履歴に必ずバージョン付きで記録する
