# 最終レポート体裁（output template）

`/analyze-race` の最終出力。`data/races/<race-id>/report.md` にも保存する。

> **v5.0（論理ファースト・表中心）**: ユーザーは**表しか見ない**前提で組む。**% は一切出さない**（発揮能力・1着率・勝率レンジ・複勝率・期待着順・条件付き% は撤廃）。
> 着順の並びは**論理**（相変位再帰の因果＝序盤A_early→中盤A_cruise→終盤A_finish）で決め、**印 ◎◯▲△× と行順**だけで強弱を伝える。
> 数字の代わりに **展開感度・好材料・懸念点** を厚く書く（**観点タグ＋なぜそう読んだか**）。これが判断の根拠。
> §2 が成果物1「展開予想」、§3 が成果物2「着順予想」。それぞれ冒頭に**検証契約**を持ち、レース後に別個採点する。
> **市場（オッズ・人気・妙味・EV・買い目）は一切出さない**（証拠にもログにもしない）。馬券選択は人間が行う。
> `tools/score_race.py` は**任意のサニティチェック**（並びの内部整合確認）に降格。出力の % は**レポートに転記しない**。

#### 各パターンの段階フロー（1行・トリガー→序盤→中盤→終盤→結果）

> 各パターンの phase_flow を**1行の因果文**で書く（トリガー → 序盤どの脚質が前/後 → 中盤で誰が消耗/温存 → 終盤どの相の能力が効いて誰が浮く/沈む → 結果の並び）。これが §3 展開感度・展開列の素。**図(mermaid)は使わない**（端末でそのまま読めるテキスト1行に統一＝点③「流れを見せる」を最小行数で）。

- **β 平均ミドル・好位差し（本線★★★）**: ⑪⑬が併走し前が締まる → 中盤で逃げ・先行が脚を使い → 終盤は余力を残した好位差し⑭⑰が抜ける（結果 ◎⑭→◯⑰→▲⑯）

- **隊列（最有力β）**: 序盤先頭 `⑪⑬④` → 最終コーナー前方 `⑪⑬⑭⑫⑯` ＋好位 `⑰⑥`
