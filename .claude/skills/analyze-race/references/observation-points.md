# 観点カタログ（observation points / factors）

調査の最小単位 = 観点。各観点は**専属 subagent**（`.claude/agents/obs-<id>.md`）が担当し、
全出走馬についてその観点の「好材料 / 懸念」と寄与スコアを構造化して返す。

> **このファイルの役割**: 観点の*カタログ的定義*（因子/相別マッピング・グルーピングの基準）＝**概念の正本**。
> 実際の調査手順・クエリ・スコア指針は各 **agent 定義**（`.claude/agents/obs-*.md`）に、全観点共通のスキーマ・規律は `research-protocol.md` にある（三者で重複させない）。

モデルの因子へのマッピング（v3.0：展開予想と着順予想を独立した2成果物に）：

| 因子 | 観点 |
|------|------|
| 潜在能力 | A, B, C |
| 適性（コース/距離/馬場） | D |
| 展開証拠（合成は STEP4a が担当） | E |
| 状態 | F, G, H, K |
| 割引（着順の取りこぼしリスク） | I |
| 条件実績（リピーター・条件付き通算パターン） | L |
| 潜在能力の代理変数（**新馬戦専用**＝厩舎新馬型・初戦準備度・出自） | M |
| 潜在能力（格の正規化・**地方競馬NAR専用**＝クラス格・転入換算の共通ランク） | N |

### v4.0：観点 → 相別能力の対応（scoring-model.md §2）

v4.0 の着順エンジンは能力を**相別4値**に分解する。**新規調査は不要**＝同じ観点 A〜K の証拠＋スクレイパ値
（`fetch_racecard.py` の `ten_speed`/`style`/`agari_best`/`recent[]`）を**相別に再バケツ化**するだけ：

| 相別能力 | 何を測るか | 寄与する観点＋スクレイパ |
|----------|-----------|--------------------------|
| **A_early** | 序盤の位置取り（テンの速さ・行き脚） | E(脚質/テン速)・K(出方)・枠 ＋ scraper `ten_speed`/`style` ＋ **重量 pace タグ（重斤量の先行＝二の脚↓）** |
| **A_cruise** | 中盤の楽な追走・位置キープ（折り合い） | C(スタミナ血統)・B(近走位置) ＋ scraper `recent[]`(位置の安定) ＋ **重量 pace タグ（重斤量＝競り合いで消耗↑）**。I(リスク全般)はA_classのdiscで効かせる（気性以外のリスクが巡航力を汚染するため分離） |
| **A_finish** | 終盤の決め手（上がり） | B(決め手) ＋ scraper `agari_best`(相対) |
| **A_class** | 相非依存の地力（全相に掛かる変調乗数） | A・B・C(base)・D/L(apt)・F/G/H/K(cond)・I(disc) ＝ 旧 ability0 を圧縮 |

> **重量（斤量・馬格）の入り口**＝`tools/weight_adjust.py` の決定論タグ（spawn 注入）。**斤量×先行は A_early/A_cruise**（二の脚↓・競り消耗↑＝展開合成「先行勢の質」で共倒れティアを増幅）、**斤量増の減点は I**、**馬格×芝ダ/馬場のパワー適性は D/G**。絶対kgの線形でなく**実効標準（性別中立化後の中央値）からの上振れ×条件（馬場/距離/先行が競るか）**で効かせる。**定量/別定では牝を性別手当ぶん中立化＝性別差は信号にせず、年齢（古馬 vs 軽量3歳）と別定加増だけを残す（混合定量で牡58を一律「重」化する誤検知を回避）。ハンデは実斤量が信号**（福島3R 20260627＝古馬58先行の共倒れを符号化）。馬場・芝ダ・距離込みで決まる＝web 再調査しない（I10）。

> 観点担当エージェントは従来どおり A〜K の評価（好材料/懸念）を出すだけでよい。相別への振り分けは scoring-model.md §2（因果骨格）が定義する。

> **v5.0（論理ファースト）の使われ方**: 各観点の好材料/懸念は、レポート §3 の **好材料/懸念点列に観点タグ付きで転記**される。
> タグ表記は **[A]指数 [B]近走 [C]血統 [D]適性 [E]展開 [F]調教 [G]馬体ローテ [H]気配 [K]騎手 [I]リスク [M]新馬 [N]格**（M は新馬モード・N は NAR モードのみ）。
> 各項は「**[タグ] 事実 ＋ なぜそう読んだか**」の形で書く（数値の代わりに根拠を厚く）。
> 相別マッピング（A_early と A_finish を別物として扱う）は、レビューで「能力読み＝A」を是正する際の主たる調整点（`review-prediction` STEP5）。
>
> **隠れ素質馬のクロスルール（読み筋・2026-06-12 追加）**: 単観点では見えない組合せを STEP4b 合成で拾う＝
> **B「凡走の敗因が展開・不利」× C「血統は当該条件適合」× E/展開合成「今回は向く展開パターンがある」**が揃った馬は、
> 着順面（実績なし）だけで切らず**該当パターンの展開列 `△`（条件付き一発）で残す**。根拠は §3 好材料に [B]敗因＋[C]血統で明記。
> 3条件が揃わない場合（例: 力負け×血統適合）は通常評価のまま。
>
> **A_finish 保険ルール（読み筋・2026-06-07 レビュー由来）**: メンバー中 **上がり最速1〜2位（A_finish 最上位級）の馬は、位置取り（E）が不利でも × や無印で全消ししない**＝最低 △ でどこかのパターン（ハイ/前崩れ系）の圏内に残す。
> 根拠: 「A_finish 上位を E を理由に切った馬が馬券内」が4レースで再発（h09④×→1着・h10①×→1着・h11⑦無印→3着・t10⑬無印→2着、いずれも 20260607）。
> 位置の不利は「届く展開が限られる」＝展開列の条件付き（`△`）で表現するのが正しく、候補からの除外で表現しない。

> v3.0 の要点：
> - **展開（観点E）は「証拠収集器」**。脚質・テン速・先行争い・バイアス・枠の**生証拠だけ**を集める。
>   確率付きパターンの構築は並列観点ではなく **STEP4a 展開合成（pace-synthesis.md）**が全証拠を突き合わせて行う。
> - **市場（旧観点J＝オッズ・人気）は撤廃**。オッズ・人気・他人の予想（専門紙の印・予想展開記事）は
>   証拠にもログにも一切使わない。予測は馬の内在情報のみで完結する。

## 純粋情報の原則（v3.0）

全観点が守る証拠の純度ルール：

- **採用＝純粋情報**: 能力指数・時計、近走内容、血統、コース/距離/馬場実績、脚質・テン速・枠、
  調教（時計・動き）、ローテ・馬体、パドック・気配、リスク要因、騎手実績・乗り替わり。
  さらに **関係者・厩舎・騎手のコメント（戦法・出方＋自信度・勝負気配）**、
  **媒体の主観評価（調教S評価・パドック点数など、馬の状態の観察）**も純粋情報として採用してよい。
- **排除**: オッズ・人気順・オッズ変動、他人の予想・印・予想展開記事。これらは群衆の意見であり証拠にしない。
- **取得元**: 脚質・テン速・枠・血統など核データは `tools/fetch_racecard.py`（keibalabスクレイパ）が公認ソース（`references/scraping.md`）。市場は引き続き取得しない。

## 観点一覧（既定 11 観点＋新馬専用 M＋NAR専用 N）

### A. 能力指数・時計 〔潜在能力〕
スピード/タイム指数、上がり3F、走破時計、レースレベル。
持ちタイムだけでなく「どのレベルのレースで出した時計か」を重視。

### B. 近走内容・クラス通用度 〔潜在能力〕
着順/着差、敗因（展開・不利・力負け）、昇級/降級時の通用度、底を見せたか。
着順より「内容」。展開不利での好走、楽勝での底を見せていない馬を評価。
**近走の実際の脚質・位置取り**も記録する（STEP4a 展開合成が E の脚質読みと突き合わせる素材）。

### C. 血統 〔潜在能力・適性両面〕
父/母父の距離・馬場・成長型、母系の活力。
特に距離延長/短縮・初芝初ダート・道悪など、実績の少ない条件の裏付けに使う。

### D. コース/距離/馬場適性 〔適性〕
芝ダ・右左回り・坂・道悪・特定競馬場での実績。
当該コース替わり/距離替わりがプラスかマイナスか。前残り傾向・内外バイアスの裏付けも（STEP4a の素材）。

### E. 展開証拠の収集 〔展開証拠〕
**この観点は確率もパターンも出さない。** 出走馬全体を見て、展開合成（STEP4a）の素になる**生証拠だけ**を集める。
出力は次を**構造化**して返す（自由記述に埋めない）：

1. **脚質分類表**: 全馬を 逃げ / 先行 / 差し / 追込 に分類。テン速・想定位置も付す。
2. **先行争いの当事者**: ハナを主張しうる馬・引きそうな馬を名指し（＝ペースの分岐点）。
3. **馬場バイアス**: 内/外・前/差し、有利不利を馬番で。コース坂形状・どのフェーズで効くか。
4. **枠の有利不利**: 枠順から見た位置取りのしやすさ。

> 確率付きの**ペースパターン構築・隊列・脚質別有利不利・反証条件は STEP4a が作る**（pace-synthesis.md）。
> E はあくまでその一次証拠。各馬の評価(-2..+2)は付けず、事実の収集に徹してよい。

### F. 調教・厩舎仕上げ 〔状態〕
追い切り時計/動き、調教場所（坂路/ウッド/南W）、本数、厩舎の勝負気配。
**媒体の調教評価（S評価等）も観察情報として採用可**（ただし時計・動きの事実を優先）。

### G. ローテ・馬体・間隔 〔状態〕
休み明け/叩き何走目、レース間隔、距離延長短縮、馬体重増減、輸送（遠征）。

### H. 当日気配・パドック・直前情報 〔状態〕
パドック・返し馬・発汗・気配、**関係者・騎手のコメント（戦法・自信度・勝負気配）**。
媒体のパドック点数も観察情報として採用可。
**直前オッズ・人気・オッズ変動は使わない（市場情報のため排除）。**
**web取得が不確実になりやすい**。取得できない場合は確信度「低」で空欄可、ユーザー補強を促す。

### I. リスク/割引要因 〔割引・減点〕
出遅れ癖、気性難、揉まれ弱さ（頭数適性＝多頭数で包まれると脆い/少頭数・外枠で一変）、故障/脚部不安、斤量増、不利枠、近走下降基調、初めての輸送/環境。
**この観点だけスコアは原則マイナス方向**（減点要因の洗い出し）。

### K. 騎手・乗り替わり・コンビ相性 〔状態〕
今走の鞍上と前走からの**乗り替わり方向**、騎手の当該コース/距離・近況の調子、馬×騎手の継続騎乗実績。
- **乗り替わり方向**: 強化（格上騎手・有力への乗り替わり）= プラス／弱化（格下・主戦離脱）= マイナス。
- **継続騎乗**: コンビ実績が良い継続は加点。テン乗り（初コンビ）は割引、特に難しい馬・先行争いが鍵のレースで重く見る。
- **騎手の状態**: 当該開催の調子（連対率・勝ち星）、当該コースの巧拙、減量(▲△☆)騎手の斤量利と経験不足の両面。
- **騎手の出方**（行く/控える）は STEP4a 展開合成の素材として記録する。
このレース単位の「誰が乗るか」要因。馬の地力（A〜C）とは独立に、状態レイヤーの乗数として効かせる。

### L. 条件実績・リピーター 〔適性・状態〕（2026-06-11 新設）
直近5走より前を含むキャリア全体を**条件で切る**: 同一レース・同コース同時期の過去好走（リピーター）、
休み明け/斤量の条件付き通算成績、**季節適性（夏馬/冬馬＝施行時期と同季節の履歴を全馬・毎回チェック。反復2走以上で信号、個体実績が無ければ0＝推定で埋めない）**。
**古い実績は鮮度評価付き**（2年以上前は条件再現性と加齢で割引）。
B（直近の内容）・A（時計）・D（コース通算）が拾えない「条件一致の過去パターン」の置き場。該当なしの馬はスコア0が正常。

### M. 厩舎新馬型・初戦準備度 〔潜在能力の代理変数〕（2026-07-07 新設・**新馬戦専用**）
過去走ゼロの新馬戦で A/B/E通過順/L/h2h が全滅した穴を埋める観点。①**厩舎の新馬仕込み型**（初戦仕上げ型/緩め出し型＝
`debut-catalog.md` ①が正本・lazy 蓄積）②**初戦準備度**（入厩時期・ゲート試験・乗り込み本数・併せの充実＝厩舎型との差分で読む）
③**出自**（セリ/募集価格・兄弟の初戦成績。価格は出自の事実＝内在情報でオッズ/人気ではない・`debut-catalog.md` ③の原則で相対読み）。
F との分担: **時計と動きの質は F、準備の量と型・出自は M**（二重化しない）。**通常レースでは起動しない**。

### N. クラス格・転入換算 〔潜在能力〕（2026-07-07 新設・**地方競馬NAR専用**）
地方はクラス階梯が場ごとに違い（南関 A1〜C3・園田 A〜C…）、混合戦・JRA/地方間の転入/移籍が常態＝
**全馬を共通ランクR（`analyze-race-nar/references/nar-class-ladder.md` が正本）に正規化**し、レース内の格上/格下と
転入馬の換算（転入初戦は1段割引・2〜3戦目が本領）を出す。JRA で `fetch_racecard.py` の `class_rank`/`class_move` が
機械でやっていた仕事の NAR 版。B との分業: **B は近走の「内容」、N は「格の位置づけ・換算」**（二重化しない）。
相別マッピング: **A_class の base 第一級項**（エンジンは `profile:"nar"` で base=N/B/A/C に組み替え＝scoring-model.md v4.7）。
**JRA レースでは起動しない**。

## グルーピング設定（重い場合の削減）

既定は 11 観点（A〜I, K, L）＝対応する 11 個の観点 subagent を並列起動。
**subagent はマージできない**ので、削減は「**起動する観点 subagent を絞る（取捨選択）**」に読み替える。下の 5/7観点グループは「どの観点が最も分離不要か＝省くならこの単位で」の目安。
並列 subagent は安価（同時最大 ~16）なので、頭数が多くてもまず 11観点を基本とし、**少頭数(〜10頭)や情報が薄いレースのときだけ絞る**：

- **5 観点モード**:
  1. 地力 = A + B
  2. 血統・適性 = C + D
  3. 展開証拠 = E
  4. 状態 = F + G + H + K
  5. リスク = I

- **7 観点モード**: A+B を統合、F+G を統合、K は状態に同梱（F+G+K）、他は単独（C, D, E, H, I）。
  騎手要因を重視したいレースでは K を単独観点として切り出す。

- **L の取捨**: 5/7観点モードでは **L は原則省く**（条件実績が薄い下級条件・若馬戦では拾い物が少ない）。
  ただし**リピーター色の濃いレース**（ハンデ重賞・季節レース・同舞台の重賞・高齢馬多数）と**盛夏（6〜9月）・厳冬（12〜2月）の開催**（季節適性＝夏馬/冬馬の信号が効く時期）では、絞る場合も **L を残す**ことを推奨（K と同じ扱い）。

- **新馬モード（新馬戦＝過去走ゼロ・観点セット自体を差し替える）**: **C, E, F, I, K, M** の6観点（＋当日再分析時のみ H）。
  **A/B/D/G/L は起動しない**（時計・近走・コース実績・ローテ・条件実績が構造的に存在せず全馬スコア0の空振りになる）。
  D を落とす代わりにコース形状（course-geometry）は従来どおり E と展開合成へ注入。各観点は agent 定義内の**「新馬戦モード」節**に従う
  （C=仕上がり早・兄弟が主証拠に昇格／E=脚質は全馬推定＝確信度低／F=主信号・水準比較／I=ゲート・気性・緩さ／K=起用の格・テン乗り割引なし／M=新設）。
  相別マッピング: M は **A_class の base**（エンジンは `profile:"debut"` で base=C/M/F に組み替え＝scoring-model.md v4.6）。

- **NARモード（地方競馬＝`/analyze-race-nar` が使用・観点セットの正本はそのスキル）**: 既定 9観点 **A,B,C,D,E,G,I,K,N**・
  速報 6観点 **A,B,D,E,I,N**。**N は NAR 必須**（格の正規化＝地力比較の第一軸）、F/H/L は情報が厚いレース（南関・ダートグレード）だけ任意追加。
  コース形状は `nar-course-geometry.md` を注入（JRA カタログでなく）。

`analyze-race` 実行時に頭数や情報量に応じて選択する。少頭数(〜10頭)や情報が薄いレースは
5〜7観点、多頭数(16〜18頭)やGⅠなど情報が厚いレースは 11 観点を推奨。**新馬戦は頭数に関わらず新馬モード**。
**地方競馬（NAR）は `/analyze-race-nar` ＝NARモードの観点セット**。
**いずれのモードでも展開合成（STEP4a）は必ず実施する**（証拠を束ねてパターンを作るのは別ステップ）。
