# 観点カタログ（observation points / factors）

調査の最小単位 = 観点。各観点は独立した調査サブエージェントが担当し、
全出走馬についてその観点の「好材料 / 懸念」と寄与スコアを構造化して返す。

二層モデルの因子へのマッピング：

| 因子 | 観点 |
|------|------|
| 潜在能力 | A, B, C |
| 適性 | D, E |
| 状態 | F, G, H, K |
| 割引 | I |
| 市場 | J |

## 観点一覧（既定 11 観点）

### A. 能力指数・時計 〔潜在能力〕
スピード/タイム指数、上がり3F、走破時計、レースレベル。
持ちタイムだけでなく「どのレベルのレースで出した時計か」を重視。

### B. 近走内容・クラス通用度 〔潜在能力〕
着順/着差、敗因（展開・不利・力負け）、昇級/降級時の通用度、底を見せたか。
着順より「内容」。展開不利での好走、楽勝での底を見せていない馬を評価。

### C. 血統 〔潜在能力・適性両面〕
父/母父の距離・馬場・成長型、母系の活力。
特に距離延長/短縮・初芝初ダート・道悪など、実績の少ない条件の裏付けに使う。

### D. コース/距離/馬場適性 〔適性〕
芝ダ・右左回り・坂・道悪・特定競馬場での実績。
当該コース替わり/距離替わりがプラスかマイナスか。

### E. 展開・脚質 × 枠・ペース 〔適性〕
想定ペース（逃げ・先行頭数）、隊列、トラックバイアス、枠の有利不利。
**この観点は出走馬全体の脚質構成を見て展開図を描く**（馬単独でなく場全体）。
出力は必ず次の4点を**構造化**して返す（自由記述に埋めない。レース後の A/B/C 仕分け照合の核になる）：

1. **脚質分類表**: 全馬を 逃げ / 先行 / 差し / 追込 に分類。テン速・想定位置も付す。
2. **隊列推移（3フェーズ）**: 序盤（〜最初の3F）／ 中盤（向正面〜3角）／ 終盤（直線・坂後）の
   各フェーズで、先頭→後方の並びを馬番で示す（1行隊列図 `④⑨ — ②⑦③ — …` でも可）。
3. **想定ペース**: H/M/S と前半通過の目安（例: 前半600m≈34.5）。ハナ争いの鍵になる馬を名指し。
   坂の位置・馬場バイアスがどのフェーズで効くかも併記。
4. **反証条件**: 「こうなれば展開想定が崩れる」を1行（例: ⑨が引かずハナ主張→ハイ化で前崩れ）。

各馬の評価(-2..+2)は「この想定展開でその馬が恵まれるか/不利か」。
**前提展開が崩れると勝率も崩れる馬は「前提展開」を注記**（例: ◎はスロー前提・ハイ化で半減）。

### F. 調教・厩舎仕上げ 〔状態〕
追い切り時計/動き、調教場所（坂路/ウッド/南W）、本数、厩舎の勝負気配。

### G. ローテ・馬体・間隔 〔状態〕
休み明け/叩き何走目、レース間隔、距離延長短縮、馬体重増減、輸送（遠征）。

### H. 当日気配・パドック・直前情報 〔状態〕
パドック・返し馬・発汗・気配、直前オッズの動き、関係者コメント。
**web取得が不確実になりやすい**。取得できない場合は確信度「低」で空欄可、ユーザー補強を促す。

### I. リスク/割引要因 〔割引・減点〕
出遅れ癖、気性難、故障/脚部不安、斤量増、不利枠、近走下降基調、初めての輸送/環境。
**この観点だけスコアは原則マイナス方向**（減点要因の洗い出し）。

### J. オッズ・人気・市場勝率 〔市場〕
単複オッズ、人気、オッズ変動（前日→直前）、過剰人気/過小評価の兆し。
**他観点と独立に**市場が織り込んでいる勝率を読む（自分の評価を混ぜない）。

### K. 騎手・乗り替わり・コンビ相性 〔状態〕
今走の鞍上と前走からの**乗り替わり方向**、騎手の当該コース/距離・近況の調子、馬×騎手の継続騎乗実績。
- **乗り替わり方向**: 強化（格上騎手・有力への乗り替わり）= プラス／弱化（格下・主戦離脱）= マイナス。
- **継続騎乗**: コンビ実績が良い継続は加点。テン乗り（初コンビ）は割引、特に難しい馬・先行争いが鍵のレースで重く見る。
- **騎手の状態**: 当該開催の調子（連対率・勝ち星）、当該コースの巧拙、減量(▲△☆)騎手の斤量利と経験不足の両面。
このレース単位の「誰が乗るか」要因。馬の地力（A〜C）とは独立に、状態レイヤーの乗数として効かせる。

## グルーピング設定（重い場合の削減）

既定は 11 観点。サブエージェント数を減らしたい場合は以下にまとめる：

- **6 観点モード**:
  1. 地力 = A + B
  2. 血統・適性 = C + D
  3. 展開・枠 = E
  4. 状態 = F + G + H + K
  5. リスク = I
  6. 市場 = J

- **8 観点モード**: A+B を統合、F+G を統合、K は状態に同梱（F+G+K）、他は単独（C, D, E, H, I, J）。
  騎手要因を重視したいレースでは K を単独観点として切り出す。

`analyze-race` 実行時に頭数や情報量に応じて選択する。少頭数(〜10頭)や情報が薄いレースは
6〜8観点、多頭数(16〜18頭)やGⅠなど情報が厚いレースは 11 観点を推奨。
