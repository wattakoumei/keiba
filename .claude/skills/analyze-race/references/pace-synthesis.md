# 展開合成プロトコル（STEP4a / pace-synthesis）

> `analyze-race` の **STEP4a**。並列ファンアウト（STEP3）が出揃った後に、**1回だけ**走る統合ステップ。
> 展開は「全馬の配置から創発する統合タスク」なので、盲目の並列観点ではなく、**全証拠を突き合わせて**ここで構築する。
> 出力は **複数の名前付き展開パターン**（= analyze-race の成果物1「展開予想」の素）。各パターンは**段階フロー `phase_flow`**（序盤→中盤→終盤の能力の効き）を持つ。
> 着順（STEP4b・論理ファースト）はこのパターン群の `phase_flow` を受け取り、相別能力で並びを論理判断する。**着順側でパターンを作り直さない。**

## 入力（STEP3 の証拠を突き合わせる）

| 証拠源 | 使う情報 |
|--------|----------|
| 観点 E（展開証拠） | 各馬の脚質（逃/先/差/追）・テン速・想定位置、先行争いの当事者、馬場バイアス、枠の有利不利、コース坂形状 |
| 観点 B（近走） | 近走の実際の脚質・位置取り（Eの脚質読みと突き合わせて確定） |
| 観点 D（適性） | コース/馬場の前残り傾向・内外バイアスの裏付け |
| 観点 K（騎手） | 各騎手の出方（行く/控える）、テン乗りの不確実性、先行意識 |
| 関係者コメント | 「ハナに行く」「今回は控えて」等の戦法宣言（純粋情報・最優先で隊列に反映） |
| 出走表 | 枠順・頭数・距離・馬場状態 |
| コース物理形状 | spawn の `course-geometry.md` 該当行（初角まで／直線長／坂）＝緩急の素・`pace_factors` の `コース形状` 行 |
| 過去開催のペース署名 | **`tools/fetch_result.py history <過去開催id…>`** の素材（勝ち馬1角位置・上がり最速の着順・ρ）。同レースの例年傾向＝`pace_factors` の `例年傾向` 行。過去idは web で特定（NARは取得不確実→web補完）|
| **重量重みづけ seed** | spawn の **`tools/weight_adjust.py` 出力**（`race.front_verdict`＋per-horse の `pace` タグ）。**先行当事者の斤量から共倒れ確率を機械算定**＝下記「先行勢の質」の必須入力。馬場・距離・芝ダ込み。決定論ミラー＝web 再調査しない（[harness-invariants](../../../rules/harness-invariants.md) I10） |

> **使わない**: オッズ・人気・他人の予想（専門紙の印・予想展開記事）。展開は内在情報のみから組む。

## 矛盾解決ルール

証拠が食い違うときの優先順位：

1. **関係者の戦法宣言 > 推測**。「逃げる」と明言があればその馬を逃げに置く（テン乗りでも宣言を優先）。
2. **近走の実脚質(B) > 一般的な脚質ラベル(E)**。前走で控えた逃げ馬は「控えうる」と扱う。
3. **複数の逃げ/先行馬がいて行く保証がない**＝先行争いが不確実 → これを**別パターンに分岐**する（単一に潰さない）。
4. テン乗り・初距離・初コースの馬の出方は不確実 → その馬が絡むパターンの確率を上げ過ぎない。
5. **先行手薄を「前残り確定」と読まない（2026-06-27 福島12R 由来・脚質可変）**。確実な先行が2頭以下＝「**前が空く**」状態は、E の**前付け転換候補**（差し×内枠×中団以上×テン速歴）が空いた前を取りに来る動機が最大化する局面。前残り(α)を厚くするのと**同時に**、転換候補が前付けするパターンを1本持つ（or α 内で当該馬を先行 riser に置く）。当該馬を**×消ししない＝印の床を△〜○へ上げる**（◎には自動昇格しない）。**バイアスが強いほど差し馬の前付け確率は上がる**＝バイアスを差し全消しの根拠に使うのは逆。E が転換候補フラグを返さなくても、確実先行≤2 なら自分で同ゲートを当てて確認する。
6. **「前残り」を本線に置くには積極根拠を要求する（2026-07-03 25件レビュー由来・前残り本線バイアス）**。25件レビューで17件が「前残り本線→実際は前崩れ/差し台頭」＝系統的な過大評価。前残り(α系)を本線ティアにできるのは、①ハナ主役が同定済み（E のテン3F実測＋前走通過1角の裏取り＝単騎がほぼ確定）②確実先行≤2で競り要素なし ③先行勢の質・斤量に共倒れ要素なし（`weight_adjust` の `front_verdict` が共倒れでない）の**3条件が全て揃うときだけ**。1つでも欠ければ前崩れ/差し台頭系を対抗以上に置き、内部 `prob` の本線−対抗差を 0.15 以内に留める。**ペース水準の機械写像（H→差し・S→前残り）だけで本線を決めない**＝因果はレース固有（2026-06-27 福島9R は公式Hで前残り＝逆写像の反例）。
   - **6b. ローカル小回りの緩和（2026-07-06 7/4-5レビュー由来・ルール6の過剰補正是正）**: 福島・小倉・函館の小回り（直線≲300m・`course-geometry.md`）では条件②（確実先行≤2）を**免除**し、**①ハナ主役同定＋③共倒れなし の2条件**で前残りを対抗以上（両方が強根拠なら本線）に置いてよい。小回りは先行が複数いても隊列決定後は前グループごと残るのが実測（2026-07-05 白河特別・九州スポーツ杯・ジュライS・大沼S＝γ前残り伏線が連続実現し、ルール6導入後に逆方向のミスが4件）。逆に**前崩れ/差し台頭を本線にする側にも積極根拠を要求**（E h2h でテン速拮抗の複数頭 or `front_verdict`=共倒れ）＝どちらの方向も機械写像でなく根拠で決める。
   - **6c. 前残りでも「逃げ本体」と「2〜4番手」を分けて読む（2026-07-12 13鞍レビュー由来）**: 前残りパターンの `result`・`per_horse_fit` で**逃げ・先導本体を中心（◎）に置けるのは「楽単騎」が立つときだけ**（ルール6①の単騎ほぼ確定＋競り・被せの要素なし）。競り/主張を伴う先導は**前残りが実現しても本体は残らない**＝中心は2〜4番手の好位勢に置く。実測: 7/12 は前残り決着9鞍で、楽単騎の逃げは4勝（函館8R②・函館12R⑤・小倉1R⑩・小倉10R⑩・小倉11R⑥）、競り/主張型の先導は5頭全滅（福島10R⑩8着・福島11R⑬12着・函館11R⑧11着・小倉10R⑪9着・小倉12R⑫14着）＝残ったのは常に番手〜好位（勝ち馬9鞍中5鞍が2〜4番手）。
7. **配役チェック（2026-07-06 7/4-5レビュー由来・証拠→隊列の断絶禁止）**。パターンを書き出す前に必ず2点を機械的に検査する:
   (a) **E の `lead_contenders`・「ハナ/好位歴あり」note の馬は、最低1パターンで先導または番手に配役する**（E が正しく指摘したのに合成が隊列 head に反映しない断絶が実測: 2026-07-05 白河特別⑧＝note あり→無配役→1-1-1-1 で4着）。
   (b) **距離短縮≥300m × 軽斤量（3歳・牝・減量で実効軽い）の馬は、脚質1段前の転換仮定を最低1パターンに入れる**（ルール5 の強制適用: 2026-07-04 雲仙特別⑥＝ダ1400→1000 短縮×3歳牝53kg を追込のまま→2-2 で1着）。
   (c) **ハナ候補の網羅（2026-07-12 13鞍レビュー由来）**: seed の近走通過で**1角1〜2番手歴のある馬は全頭**、いずれかのパターンの `trigger`/`contesters` に登場させる（「主役級だけ」で trigger 群を組まない）。7/12 は trigger 候補群に居ない馬のハナが2鞍（七夕賞③＝候補⑧⑪⑬⑮⑯の外から単騎逃げ・巴賞③＝同）＋追込/差し分類馬の逃げ・先行が2鞍（小倉7R⑤・福島10R⑪）＝計4鞍で隊列前提が崩壊。E の分類ラベルでなく**通過順の事実**で候補集合を閉じる。
8. **既知事実トリガーのパターンは伏線に置かない（2026-07-12 13鞍レビュー由来・ティア規律）**: `trigger` の実現条件が**分析時点で既に観測済みの事実**（前走で行き脚不発を実測・当日発表済みの道悪・戦法宣言）だけで構成されるパターンは、**発動が「if」でなく「継続」**＝最低でも対抗★★に置く。伏線★は「複数の未観測条件が重ならないと発動しない」筋に限る。実測: 7/12 は自書トリガーがそのまま実現したパターンを伏線に置いた敗因が3鞍（阿蘇S γ「①の行き脚が戻らず」＝前走で実測済み→伏線0.32のまま1-3着完全的中・函館5R γ「重馬場で先行群が早く動く」＝重馬場は既知・七夕賞γ）。伏線実現5/13 は分布として多すぎる＝ティアの写像（内部 prob）より先にこの規律で下駄を履かせる。

## 新馬戦の合成（新馬モード・過去走なし）

新馬戦は脚質・テン速が**全馬推定**（E の素は調教のテン・関係者コメント・血統の先行力だけ＝通過順・h2h・テン3F実測ゼロ）。合成の特則:

1. **本線に寄せすぎない**: 内部 `prob` の本線−対抗差は **0.15 以内**を既定とする（隊列の確信が構造的に持てない）。「単騎逃げ」認定はコメントの戦法宣言があるときだけ。
2. **新馬固有の分岐を最低1本残す**: 出遅れ・行きたがり・ダッシュがつかず隊列が想定から崩れるパターン（発動トリガー=「スタート直後の出脚」）。矛盾解決ルール6（前残り本線の3条件）は新馬では①ハナ主役同定がまず満たせない＝前残り本線は原則不可。
3. **formation は粗くてよい**: 先頭候補群（2〜4頭）と後方候補群の2層まで。馬番を並べ切る精度は無い＝偽の精度を出さない。
4. **per_horse_fit は「位置の推定確度」込みで**: 脚質推定の確信度が低い馬は fit の絶対値を抑える（±2 を乱発しない）。

## 複数パターンの作り方

**先行争いの分岐点**を軸に 2〜4 個のパターンを立てる。各パターンは「誰がハナを取り、ペースがどうなるか」の
1つの筋書き。確率は「その筋書きが起きる尤度」。

- 典型は **平均/前残り・ハイ/差し台頭・スロー/瞬発** の3系統だが、レースに合わせて命名する（単騎逃げ濃厚なら2個でよい）。
- 各パターンに **発動トリガー**（観測可能な条件。例「⑨が引かずハナ主張」）を必ず付ける → 当日にどのパターンが起きているか判定できる。
- 各パターンに **段階フロー `phase_flow{early,mid,late,result}`** を必ず著作する（**v5.0 で必須**）。
  因果文には**コース物理形状（spawn データの `course-geometry.md` 該当行＝直線長・坂・初角）を織り込む**
  （例:「中山の急坂で甘い逃げは止まる」「新潟外658mで上がり最速勢が台頭」「東京芝2000は初角130mで外枠が位置を取れない」）。
  「序盤→A_early→中盤→A_cruise→終盤→A_finish→結果」の**因果文**で、各相で*どの能力が効いて誰が浮く/沈むか*を書く。これが report の §2 段階フロー（1行テキスト）と §3「展開感度」の素になる（点③＝流れを見せる）。
- `leg_advantage`（脚質別の有利不利符号 -2..+2）はラップ非依存の主検証軸。レース後に「実際どの脚質が好走したか」と照合する。
- 馬別に `per_horse_fit`（パターン別の恵まれ度 -2..+2）を付す。**§3 の「展開列」（各馬が各パターンで圏内か・どれだけ強いか）の源**なので、**少なくとも印が付く想定の上位馬は全頭**、各パターンでの fit を埋める（枠・位置取りで脚質ラベルから外れる馬は特に重要）。fit と各パターンの浮上/沈む馬は整合させる（食い違えばパターン側を見直す）。
- **捲り/ロンスパ型の同定（2026-07-08 読み落とし3件由来・脚質ラベル救済）**: seed の近走通過順で**2走内に道中の位置上げ実績**（向正面〜3角で4個以上前進。例 13-13-8-2）があり、決め手が**瞬発型でなく持続型**（上がり一発でなく長く良い脚を使う・スタミナ血統）の馬は捲り候補。**発動環境**＝スロー想定 or 小回り・洋芝・上がりのかかる馬場（S戦は特に）。該当馬をスロー系パターンで「差し/追込だから届かない」と機械的に消さず、`per_horse_fit` に**△（条件付き圏内＝自分で動ける）**を付けて展開列・印に載せる。当該パターンの `phase_flow` mid に「当該馬が向正面から動けばロンスパ戦へ急変（前も脚を使わされる）」の因果を書き、トリガーに「向正面で当該馬が動くか」を入れる（実測: 2026-07-05 小倉10R⑬ 13-13→3角捲り3着=展開列空／2026-06-28 函館11R② 13-12-8-2→1着／2026-06-20 阪神10R⑪ 18-17-1-1→2着）。
- **報告は確率%でなく可能性ティア（本線/対抗/伏線）**。内部の `prob`（`Σ prob = 1`）は計算・ログ用に残し、レポートには出さない。

## 展開トリガー早見（pace_factors）の著作

パターン群を作ったら、**「どの展開が来そうか」をユーザーが判断・当日に再ティアするための材料**を `pace_factors[]` に race-level でまとめる
（pattern ごとの `trigger`＝個別スイッチに対し、これは立っている材料の一覧。`report.json` の `pace.pace_factors`、web は §2 先頭に表示）。
**新規調査はしない**＝STEP3 で集めた証拠とカタログ／seed から導出する（別ファンアウト不要）。各要素 `{factor, reads, day_check}`:

- **最低限カバーする factor**（レースで足し引きしてよい）:
  1. `先行勢の数と質` — 逃げ/先行の頭数・テン速上位・「行かざるを得ない馬」の有無（観点E＋seed のテン速）。最大の決定因。
     **斤量も「質」に含める**＝`weight_adjust.py` の `front_verdict` を取り込む: **先行当事者の上位が実効標準（性別中立化後の中央値。`race.base_effective`）より重い（`level`=重/やや重）なら、①単騎でも二の脚で抜け出しにくく②競ると消耗が重い → 共倒れ(差し台頭=γ系)パターンのティアを上げる増幅入力**。性別差は中立化済み＝牝の軽さは信号にしない（効くのは年齢=古馬の斤量負担と別定加増。ハンデ戦は実斤量が信号）。
     ただし**線形でなく条件付き**＝「重い先行が"複数で競る"」ときに効く（単騎で楽に行ければ前残りは残る）。軽い先行が1頭混じるだけでは効かせない（福島3R 20260627: 58kg先行⑨⑫が競って共倒れ・55kg先行⑤は標準で残存＝この非対称をツールが符号化済み）。**斤量は脚質の質を増幅する従の因子**であって主因（先行争いの構造）に取って代わらない。
     **先行が薄い（確実先行≤2頭）ときは「前残り確定」でなく「前が空く→差し馬の前付け転換」を両読みする**（矛盾解決ルール5）。E の前付け転換候補フラグ（差し×内枠×中団以上×テン速歴）を `reads` に挙げ、`day_check` に「当該馬の二の脚・出ムチ＝前を取りに行ったか」を入れる。転換候補が前を取れば α の前残りはむしろ堅くなる（前を作る馬が増える）一方、その馬自身が圏内へ浮く＝従来の脚質ラベルで×消ししていた馬が勝ち負けになる。
     **頭数バンド**（従の符号・race-level）: **少頭数（≦10頭）**＝隊列が固定しやすく単騎逃げ・スロー系を必ず1本残す。揉まれ弱い馬（観点Iの非決定論層）の懸念は解除方向＝`per_horse_fit` を下げすぎない。**多頭数（≧16頭）**＝先行争い・外々ロス・包まれのリスクが顕在化＝揉まれ弱い馬の fit を密集パターンで下げる。11〜15頭は補正なし。
     **捲り候補（同定は「複数パターンの作り方」）がいるレースはスロー系パターンが不安定**＝当該馬が向正面で動けばロンスパ戦へ急変（前も後ろも消耗）。`reads` に候補馬と「動けばスロー前残りは崩れる」の条件文を書く（発走後の事象なので `day_check` は不要・パターン側トリガーで持つ）。
  2. `枠の効き` — 内/外の先行馬の主張しやすさ・外々ロス（確定枠を先取り＝I6）。
  3. `コース形状` — 初角までの距離・直線長・坂（`course-geometry.md`）＝緩急と前残り/差し届きの素。
     **距離バンド補正**（形状の上に重ねる従の符号。**形状で説明できない残差だけ**＝二重計上しない）:
     - **〜1400（短）**: 絶対速度が高く隊列が即決＝位置（A_early）の価値↑・差し/追込を一段重く割引。`reads` は「平均でもα(前残り)を一段上げ／差しは展開待ち」。
     - **1600〜2000（中）**: 補正なし＝コース形状をそのまま出す。
     - **2200〜（長）**: 道中が緩みやすく分散大＝スロー前残り(γ)とハイ消耗(β)を**両方潰さない**・A_cruise/スタミナの価値↑。`reads` は「両ティアを残し1つに固定しない」。
     ※これは `reads`（定性・条件文）に書く読み筋であって `leg_advantage` を手で固定する数値ではない（I2 %禁止・「脚質別有利不利を手で固定しない」を維持）。
  4. `例年傾向（過去開催）` — `fetch_result.py history` のペース署名（勝ち馬1角位置・上がり最速の着順・ρ）から**定性傾向**を書く（前残り基調／差し決着多め）。過去idは web で特定、NARはweb補完。**結論H/M/Sは出さない**。
  5. `当日で動く点` — 馬場含水・前半参考Rのバイアス・**風（強風日のみ効かせる: 直線向かい風＝前が失速しやすく決め手勢台頭へ／直線追い風＝前残り助長。向正面の追い風はテンが速くなりやすい）**等、当日にしか分からない要素（`§0 day_board` と連動）。
- **`reads` は定性**（% 禁止＝I2）。脚質別有利不利を手で固定せず「○○なら β 寄り／△△なら α 固定」と**条件→パターン**で書く（当日ティア付け替えの素）。
- **`day_check` が埋まる factor は `§0 day_board` にも観察枠を作る**。当日採取→`counter_conditions` でティア付け替え／1パターン固定（当日可変と一本の線にする）。

```json
      "name": "平均ペース・前残り",
      "prob": 0.50,

      "trigger": "⑨が控え単騎④が緩めに逃げる",
      "first_600m": "35.0前後",
      "pace_level": 0.45,
      "contesters": [4],
      "leg_advantage": {"逃げ": 1, "先行": 1, "差し": -1, "追込": -2},
      "formation_head": [4, 9],
      "formation_last_corner": [4, 2, 9, 5],
      "bias": "前/内有利",
      "phase_flow": {
        "early": "④が単騎で楽にハナ・全体が密集して位置取りが安い（A_early が効くが消耗は小）",
        "mid": "緩いペースで誰も脚を使わず前が余力を温存（A_cruise が効く）",
        "late": "位置リードを保ったまま瞬発勝負・前にいた④②が止まらない（A_finish×位置）",
        "result": "前残り＝④②浮上、差し⑤・追込勢は届かず"
      },
      "per_horse_fit": [{"no": 5, "fit": -1}, {"no": 10, "fit": 1}]
    }
  ],
  "falsification": "⑨がハナ主張すれば β(ハイ)の確率が 0.35→0.6 へ。逆に誰も主張しなければ γ(スロー)上昇。",
  "field_note": "逃げ2・先行3。鍵は⑨の出方。テン乗り⑩は読み難く各パターンで中立寄り。"
}
```

- `pace_level` (0..1): **v4.0 で必須**。スコアラーの3相再帰へ渡すペース水準。ラベルから決める：
  `超スロー 0.1 / スロー 0.25 / 平均 0.5 / やや速 0.65 / ハイ 0.85 + 0.15*(先行争い当事者数 - 1)`（clip 0..1）。`first_600m` があれば整合させる。
- `contesters`: ハナ〜番手を争う馬番リスト（E の先行争い当事者）。`pace_level` の当事者数項と、非先行馬の位置抑制に使う。
- `formation_head`: 序盤の先頭2〜3頭（馬番）。netkeiba の通過順と照合可能な粒度。**著作値が正本**。空なら `score_race.py` が序盤後 pos から逆生成して埋める。
- `formation_last_corner`: 最終コーナーの前方隊列（馬番、前→後）。同上、著作優先・無ければ engine 逆生成。
- `leg_advantage` と `formation_*` が **ラップが取れなくても着順・通過順から検証できる**指標（検証軸の正本）。
- `likelihood_tier`: **本線 / 対抗 / 伏線**（報告用。偽の精度を避けるため%でなくティアで出す）。内部 `prob` から写像（最大prob=本線、以下対抗・伏線）。
- `phase_flow{early,mid,late,result}`: **v5.0 で必須**。各相で*どの能力が効いて誰が浮く/沈むか*の因果文。§2 段階フロー（1行）・§3 展開感度の素。
  early=序盤の位置取り（A_early が効く相）、mid=中盤の消耗/温存（A_cruise）、late=終盤の決め手（A_finish×余力）、result=結果隊列。

## STEP4b への受け渡し（v5.0・論理ファースト）

- **着順の並びは論理で決める**（SKILL STEP4b）。最有力パターンの `phase_flow` に各馬を通し、相別能力（A_early/A_cruise/A_finish/A_class）で
  どの相で浮く/沈むかを判断して **印 ◎◯▲△× と行順**を付ける。**並びの強弱に%は使わない**（散文%禁止）。単勝率/複勝率の2カラムは `score_race` から別途 rank に注入する（v5.1・並びは論理が主＝率順に並べ替えない）。
- `phase_flow` が **§3「展開感度」** の直接の素。`leg_advantage`/`formation_*` は**展開検証の正本**として保持。
- **率の源＋並びのサニティ（v5.1）**: `tools/inject_probs.py`（内部で `score_race.py`）が観点スコア＋このパターン群から **単勝率/複勝率を毎回算出し rank[] に注入**する。
  その**並び順**が論理の並びと食い違わないかも確認する（食い違えば論理側を正とし理由を残す＝率順に並べ替えない）。エンジンが返す派生 leg_advantage/formation は整合チェック用。
- scoring-model の相別能力・3相再帰は**並びの論拠の語彙**として使う（数式は因果骨格）。同じ数式から出る %（win_prob/place_prob）は**率2カラムとして rank に載せる**が、**並びは論理が主**（率は参考列）。

## 当日可変（SKILL STEP6 から呼ばれる）

当日情報（前半参考Rで採取したバイアス・パドック・乗り替わり・馬場変化・取消・雨・強風・テン乗り解消など）を受けたら：

**当日バイアスの採取（参考レース選定基準）**: 本命は後半R中心なので、前半の同条件Rを観察して内外・前後のバイアスを採る（当日カードは `tools/fetch_racecard.py day` で取得）。
参考Rの一致は **芝/ダート（必須・絶対に混ぜない）＞ 同日・時間帯（必須・直前ほど重い）＞ 回り内外・右左 ＞ 距離帯** の優先順位。
回り・距離は完全一致が理想だが無ければ近いもので代用し**割り引く**（距離違いは決まり手・伸び位置のみ流用）。**1鞍で断定せず複数Rの傾向**で見る（内有利は開催進行で外有利へ反転しうる）。

**同日同場の確定済みレースは最優先の参考R＝発走前チェック必須（2026-07-11 14鞍レビュー由来）**: 朝にまとめて分析した日でも、**発走前に同場の確定済みRの決着傾向（実効ペース層・決着脚質＝前残りか差しか）を確認し、この当日可変を1回回してから発走を迎える**（朝の分析のまま放置しない）。同方向の決着が**同条件で2鞍以上連続**していれば下の非対称ルールの「複数鞍」を満たす＝本線の降格・対抗の格上げまで可。
根拠: 0711小倉は6R以降H差し決着が3連続（6R→10R→11R）だったのに後半3鞍のティアが朝の分析のまま＝差し側の対抗βが3鞍とも実現（小倉11Rは浮沈リストほぼ完全的中）してティアだけ外した。

1. **再調査はしない**（パターン構造は維持）。
2. 当日情報に基づき **可能性ティアを付け替える**（例: 伏線→本線へ格上げ）、または「このパターンで確定」と**1つに固定**する。
   - 例: 「⑨が大外枠で楽にハナ→ β(ハイ) を本線へ格上げ・α を対抗へ」「馬場が急速に乾いて前残り→ α を本線へ」。
   - **非対称ルール（2026-06-07 レビュー由来）**: 参考Rを根拠にティアを**下げる**（本線/対抗→伏線）には、**同条件（芝/ダ・距離帯一致）の複数鞍**で同じ傾向が必要。**1鞍だけなら「上げ」までに留め、既存パターンは下げない**。
     根拠: 安田記念(t11)は芝1600・1鞍の差し決着で α(前残り)を伏線へ下げ→実現はα。逆に h12 は同条件ダ3鞍で P1 固定→◎▲△で完全的中。馬場バイアスは参考Rから移せるが、**ペース構造はレース固有**＝他レースの決着型だけで既存の筋を消さない。
     **本線の追認・強化も同様（2026-07-03 追記）**: 1鞍を根拠に本線の確度を上げたり対抗を下げたりしない（2026-06-28 小倉10R: 参考1鞍の前残りで α 追認・β 格下げ→実際は真逆のH差し総取り・格上げした◯が16着）。
3. 最有力パターンが変わったら、その `phase_flow` に沿って **§3 の展開感度と並び（印）を論理で再評価するだけ**（素の能力読み＝好材料/懸念点はそのまま）。
   エンジンを併用する場合は新しい `prob`/`pace_level` で再実行し並びの整合だけ確認する。
4. ただし当日情報が**特定馬の状態**（パドック激減・腹目立つ等）なら、その馬の観点 H/G の読み（好材料/懸念点）だけ更新して並びを部分見直ししてよい。

> 当日更新は **report.json を更新 → `tools/validate_report.py <race-id>` → `tools/project_predictions.py <race-id> --update`**（predictions.jsonl に `note:"当日更新"` で履歴 append・上書きせず履歴を残す）。
