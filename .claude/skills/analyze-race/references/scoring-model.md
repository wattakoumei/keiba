# スコアリングモデル（発揮能力 → 展開パターン → 着順分布）

> **このファイルが唯一の調整点。** 改善ループ（review-prediction）はここの重み・閾値を編集する。
> 変更時は末尾の「変更履歴」にバージョンと理由を追記すること。
>
> **目的（v3.0〜）**: このハーネスは **展開予想と着順予想の2つを独立した成果物**として精度に全振りする
> 純予測エンジン。妙味・EV・Kelly・買い目は持たず、**市場（オッズ・人気）も一切使わない**（証拠にもログにもしない）。
> 証拠は馬の内在情報のみ。展開は §7 の「展開合成（STEP4a / pace-synthesis.md）」が作る**複数の名前付きパターン**で表し、
> 着順はそのパターン群で条件づけて算出する。

## モデルバージョン: v3.0

各馬について、観点 A〜K（市場Jを除く）の評価（-2..+2）を集約し、**展開パターンで条件づけた着順分布**を出す。
パイプライン：「ペース中立の素の力（§2-6）→ 展開パターンで条件づけ（§7）→ パターン加重の着順分布（§8-9）」。

### 1. 観点評価の取り込み

各観点の馬ごと評価 `e_X ∈ [-2, +2]` を 0..1 に正規化：`n_X = (e_X + 2) / 4`。
（確信度「低」の観点は中央 0.5 方向へ 30% 収縮：`n_X ← 0.5 + (n_X - 0.5)*0.7`）

### 2. 潜在能力 base（0–100）

A・B・C の加重平均を 0–100 にスケール。

| 観点 | 重み |
|------|------|
| A 能力指数・時計 | 0.40 |
| B 近走内容・クラス通用度 | 0.40 |
| C 血統 | 0.20 |

```
base = 100 * (0.40*n_A + 0.40*n_B + 0.20*n_C)
```

### 3. 適性 multiplier（≈0.85–1.15）

**D のみ**（コース/距離/馬場）を中立 1.0 まわりに展開。`n` が 0.5 で 1.0。
※ 展開（脚質×ペース）は適性に含めず、§7 のパターン条件づけで効かせる。

```
apt = 1.0 + 0.15*(2*n_D - 1)   # ≈ [0.85, 1.15]
```

### 4. 状態 multiplier（≈0.77–1.23）

F・G・H・K を中立 1.0 まわりに展開。

| 観点 | 効き幅 |
|------|--------|
| F 調教・厩舎（媒体の主観評価も可） | ±0.07 |
| G ローテ・馬体・間隔 | ±0.05 |
| H 当日気配・パドック（媒体の点数も可） | ±0.05 |
| K 騎手・乗り替わり | ±0.06 |

```
cond = 1.0 + 0.07*(2*n_F - 1) + 0.05*(2*n_G - 1) + 0.05*(2*n_H - 1) + 0.06*(2*n_K - 1)  # ≈ [0.77, 1.23]
```

> 関係者・厩舎・騎手のコメント（戦法・出方＋自信度・勝負気配）は純粋情報として F/H/K に取り込む。
> ただし**直前オッズ・人気・他人の予想は使わない**（観点 H からも市場情報を排除）。

### 5. 割引（減算・着順の取りこぼしリスク）

I のスコア `e_I ∈ [-2, 0]` を base に対する減点に変換。出遅れ癖・気性難・脚部不安・距離不安・大不利枠など、
能力はあっても着順が安定しない馬を厚めに引く。

```
discount = base * 0.10 * (-e_I)    # e_I=-2 で base の20%減
```

### 6. ペース中立の素の発揮能力

```
ability0 = base * apt * cond - discount
```

ability0 が負になる場合は 0 でクリップ。**ここまではペースを織り込んでいない「素の力」**。

### 7. 展開パターンによる条件づけ（着順精度の心臓部）

展開は観点 E が単独で出すのではなく、**STEP4a 展開合成（pace-synthesis.md）**が全観点の証拠
（E の脚質・テン速・先行争い、B の近走脚質、D のバイアス、K の騎手の出方、関係者コメント、出走表の枠）を
突き合わせて作る **複数の名前付きパターン**で表す：

```
patterns = [
  { id:'α', name:'平均・前残り',     prob:0.50, leg_advantage:{逃げ:+1,先行:+1,差し:-1,追込:-2}, trigger:'⑨が控える' },
  { id:'β', name:'ハイ・差し台頭',   prob:0.35, leg_advantage:{逃げ:-2,先行:-1,差し:+1,追込:+1}, trigger:'⑨がハナ主張' },
  { id:'γ', name:'超スロー瞬発戦',   prob:0.15, leg_advantage:{逃げ:+1,先行:+1,差し:0, 追込:-1}, trigger:'誰も行かず' },
]   # Σ prob = 1
```

各パターン p について、各馬の脚質と想定位置取りからペース適合 `pace_fit_i,p` を掛けて
**パターン条件づき発揮能力**を作る：

```
ability_i,p = ability0_i * pace_fit(脚質_i, 位置_i, p)
```

`pace_fit` の目安（中立 1.0、効き幅 ±0.15 程度。パターンの `leg_advantage` 符号と `per_horse_fit` を使う）：

| 有利不利の符号 | -2 | -1 | 0 | +1 | +2 |
|----------------|----|----|---|----|----|
| pace_fit 係数  | 0.86 | 0.93 | 1.00 | 1.07 | 1.14 |

> 位置取り（隊列の前後・内外・包まれ）で ±0.03 程度さらに微調整してよい。
> pace-synthesis が馬別に `per_horse_fit` を返す場合はそれを上表より優先する。
> **どのパターンでも崩れない自在馬は全パターン中立（0）**にする。

### 8. パターンごとの勝率 → パターン加重で着順分布

各パターン p で softmax により勝率シェアに変換（温度 `T`）：

```
T = 14
p_i,p = exp(ability_i,p / T) / Σ_j exp(ability_j,p / T)     # = パターン p の条件付き勝率
```

**最終の自分の勝率**はパターン確率で加重平均：

```
p_i = Σ_p prob_p * p_i,p
```

`p_i` が「自分の勝率」%。**各パターンの条件付き勝率 `p_i,p` は潰さず保持し**、レポートとログに併記する
（output-template §3・§4。「もしこのパターンなら着順はこう」を見せ、当日に付け替え可能にするため）。

### 9. 着順分布（複勝率・連対率・予想着順）

Harville 近似で各パターンの勝率 `p_i,p` から着順分布を作り、パターン加重する。

```
P(iが1着) = p_i
P(iが2着) = Σ_p prob_p * Σ_{k≠i} p_k,p * p_i,p/(1 - p_k,p)
P(iが3着) = Σ_p prob_p * Σ_{k≠i}Σ_{l≠i,k} p_k,p * p_l,p/(1-p_k,p) * p_i,p/(1-p_k,p-p_l,p)
複勝率 p3_i = P(1着)+P(2着)+P(3着)   （※3着圏。少頭数は2着圏=連対率も併記）
```

**予想着順**: `期待着順 E[rank]_i = Σ_r r * P(iがr着)` の昇順、または複勝率降順で確定。

> **当日可変（SKILL STEP6）**: 当日情報でパターン確率 `prob_p` を付け替える、または1パターンに固定した場合、
> §8-9 を**再計算するだけ**でよい（§2-6 の素の力は再調査不要）。固定時は `p_i = p_i,固定パターン`。

### 10. 偶然（信頼区間）

点推定 `p_i` を過信しないため、ブレ幅を注記する。主因は**パターン間のばらつき**：
ペースパターンで勝率が大きく変わる馬ほど（展開待ちの馬ほど）レンジを広く取る。

```
σ_pattern_i = std_p(p_i,p)                         # パターン間の勝率標準偏差
σ_i ≈ σ_pattern_i + p_i * (0.10 + 0.10*低確信度割合 + 0.05*(頭数>14 ? 1 : 0))
勝率レンジ = [p_i - σ_i, p_i + σ_i]（0未満は0でクリップ）
```

## パラメータ早見表（調整対象）

| 記号 | 意味 | 初期値 |
|------|------|--------|
| base 重み | A/B/C | 0.40/0.40/0.20 |
| apt 効き幅 | D | ±0.15 |
| cond 効き幅 | F/G/H/K | ±0.07/±0.05/±0.05/±0.06 |
| pace_fit 効き幅 | 展開パターン（脚質×有利不利符号） | ±0.15（§7 表） |
| discount 係数 | I | 0.10 |
| T | softmax 温度 | 14 |

> 着順精度の主役は **§7 pace_fit と §2 base 重み**。review はまずこの2つを動かす。
> **市場（オッズ・人気・α）は存在しない。** モデルは馬の内在情報のみで完結する。
