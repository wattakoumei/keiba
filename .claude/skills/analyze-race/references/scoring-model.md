# スコアリングモデル（相別能力 → 3相再帰：着順の並びの**因果骨格**）

> **役割（v5.0 で変更）**: このファイルは **相変位再帰の因果骨格（相別能力 A_early/A_cruise/A_finish/A_class → 3相再帰）の定義書**である。
> v4.0 までは「着順%の正本」だったが、**v5.0（論理ファースト）でその役割を撤回**：
> - **着順の並びは論理で決める**（SKILL STEP4b）。ここの相別能力・3相再帰は、その並びを判断するための**語彙と因果の骨格**として使う
>   （§2 段階フロー・§3 展開感度がこの A_early/cruise/finish の言葉で書かれる）。
> - **`tools/score_race.py` は単勝率/複勝率(win_prob/place_prob)の源**（v5.1 で昇格）＝`tools/inject_probs.py` 経由で**毎回回し rank[] に注入**する。ただし**並び（印＋行順）は依然 論理が主**＝率はその並びに添える参考列で、率順に並べ替えない。
>   論理の並びとエンジンの並び順が食い違ったら**論理側を正**とし、理由を `header_notes`/`engine_check` に残す（食い違いは点検シグナル）。較正先(市場)の無い内在%＝偽の精度である旨を明記する（散文%は依然禁止＝率は数値Fのみ・web が×100表示）。
> - 「着順%・Brier が第一指標」という旧方針は撤回。検証は**並びの順位相関**＋**展開/材料の質的的中**が主（`review-prediction`）。
>
> 6ノブ（早見表）と `score_race.py` の `PARAMS` の同値ミラーは**維持必須**（v5.1 でエンジンが率の源になった＝毎回回るので常時効く）。
> 改善ループで「同型ミスが ≥3レース 再発」したとき 6ノブを小刻みに触る（率の分布を動かす）。変更時は `score_race.py` の `PARAMS` を必ず同値にミラーし、末尾「変更履歴」にバージョンと理由を追記。
>
> **目的**: このハーネスは **展開予想と着順予想の2つを独立した成果物**として精度に全振りする予測エンジン。
> 妙味・EV・Kelly・買い目は持たず、**市場（オッズ・人気）も一切使わない**（証拠にもログにもしない）。証拠は馬の内在情報のみ。
> 展開は「展開合成（STEP4a / pace-synthesis.md）」が作る**複数の名前付きパターン**（段階フロー phase_flow 付き）で表し、着順はその phase_flow に沿って論理で並べる。

## モデルバージョン: v4.0（相変位再帰モデル）

**設計思想（v4.0）**: 着順は「能力×ペース係数」の一発計算ではなく、**展開↔能力を相ごとに交互に効かせる再帰シミュレーション**で創発させる。
能力を**単一スカラーから相別4値に分解**し、各馬の状態 `{pos(位置), energy(余力)}` を**序盤→中盤→終盤と持ち越す**。
スローの前残りも、ハイの差し台頭も、パターンの **pace_level** から構造的に出る（脚質別有利不利を手で固定しない）。

> **なぜ v3.0 から変えたか（加古川特別の失敗）**: v3.0 は単一 `ability0`（レンジ15–115）に `pace_fit`（±15%）を掛けるだけで、
> ①正しく読んだ強バイアスを着順に伝達できない、②1個の能力値が「序盤の速さ」と「終盤の決め手」を潰す、という構造欠陥があった。
> 結果、前残りを的中させながら追込◎を勝率85%に置き（実5着）、勝った逃げ馬を△2%に過小評価。
> v4.0 は相別能力＋状態持ち越しでこの**伝達不足**を是正する。詳細は変更履歴 v4.0。

パイプライン：**相別能力（§2-5）→ 各パターンで3相再帰（§6）→ パターン別 softmax（§7）→ パターン加重の着順分布（§8-9）→ 偶然レンジ（§10）**。

### 1. 観点評価の取り込み（v3.0 から不変）

各観点の馬ごと評価 `e_X ∈ [-2, +2]` を 0..1 に正規化：`n_X = (e_X + 2) / 4`。
（確信度「低」は中央 0.5 方向へ 30% 収縮：`n_X ← 0.5 + (n_X - 0.5)*0.7`。「低〜中」は 15% 収縮 `×0.85`。）

> 観点→相別能力の対応は `observation-points.md`。**v4.0 は新規調査を増やさない**（同じ A〜K 証拠＋スクレイパ値を相別に再バケツ化するだけ）。

### 2. 相別能力（4値・各 0..1）

各馬について、観点スコア＋スクレイパ（`fetch_racecard.py` の `ten_speed`/`style`/`agari_best`/`recent[]`）から4つの相別能力を作る。
スクレイパ値のルックアップは**構造定数**（新規パラメータにしない）。

**A_early（序盤・位置取り）** — テンの速さ・脚質・騎手の出方・枠
```
ten = {速:1.0, 中:0.5, 遅:0.1}[ten_speed]          # scraper
sty = {逃:1.0, 先:0.75, 差:0.3, 追:0.1}[style]      # scraper（関係者「ハナ宣言」は sty→逃 に固定）
A_early = clip(0.55*ten + 0.35*sty + 0.10*n_K + draw_adj, 0, 1)   # draw_adj: 枠の有利不利 ±0.05
```

**A_cruise（中盤・楽に位置を保つ）** — 近走位置の安定・スタミナ血統・近走内容・気性
```
stab = 1 - clip( pstd(recent.first_corner / recent.field) / 0.30, 0, 1)   # scraper recent[]（無ければ0.5）
A_cruise = clip(0.50*stab + 0.35*n_C + 0.15*n_B, 0, 1)      # I(リスク全般)はA_classのdiscで効かせる。A_cruiseに混ぜると脚部不安等が巡航力を不当に下げる
```

**A_finish（終盤・決め手）** — 上がり最速（相対）・近走の決め手
```
af = clip( (field_max_agari - agari_best_i) / (field_max_agari - field_min_agari), 0, 1)  # 速いほど1（scraper）
A_finish = clip(0.6*af + 0.4*n_B, 0, 1)            # agari無ければ af=0.5
```

**A_class（相非依存の地力・全相に掛かる乗数）** — 旧 ability0 を圧縮し**支配項から乗数へ降格**
```
base = 0.40*n_A + 0.40*n_B + 0.20*n_C              # v3.0 §2 と同じ A/B/C 重み（0..1スケール）
apt  = 1.0 + 0.15*(2*n_D - 1)                       # v3.0 §3 と同じ（適性=D）
cond = 1.0 + 0.07*(2*n_F-1)+0.05*(2*n_G-1)+0.05*(2*n_H-1)+0.06*(2*n_K-1)   # v3.0 §4 と同じ（状態 F/G/H/K）
disc = 0.10*(-e_I)                                   # v3.0 §5 と同じ（割引 I）
A_class_raw = clip(base*apt*cond - base*disc, 0, 1)
A_class = CLASS_FLOOR + (1 - CLASS_FLOOR)*A_class_raw      # 変調帯 [CLASS_FLOOR, 1] へ圧縮
```

> **★ A_class 圧縮が v4.0 の肝**。これが無いと A_class(0..1) が終端スコア全体に掛かり、v3.0 と同じ「地力支配」へ戻る
> （加古川⑥が再び85%に化ける）。`CLASS_FLOOR=0.60` で地力を「位置と競える乗数」に留める。
> 関係者・厩舎・騎手のコメントは純粋情報として F/H/K に取り込む。**直前オッズ・人気・他人の予想は使わない。**

### 3〜5. （旧 apt/cond/discount は §2 の A_class 定義に内包）

v3.0 の §3 適性・§4 状態・§5 割引の式は、上の `A_class` の `apt`/`cond`/`disc` として**そのまま生きている**。
ただし役割が変わり、**支配的加算項ではなく、位置＋決め手の和に掛かる乗数**になった点に注意。

### 6. 3相再帰（着順精度の心臓部・決定論）

各展開パターン p について、各馬の状態 `pos_i∈[0,1]`（1=先頭側）・`energy_i∈[0,1]`（1.0開始）を
**序盤→中盤→終盤**と更新する。パターンは **pace_level `L_p∈[0,1]`** を供給する（§7、pace-synthesis.md）。

```
# --- 序盤（位置確定＋初期消耗）---
intent_i = A_early_i * (1 - 0.5*L_p)                      # ハイなほど誰もが前に行けない
pos_i    = sigmoid( k_pos * (intent_i - mean_j(intent_j)) )
energy_i = 1 - g_fwd * L_p * max(0, pos_i - A_cruise_i)   # 巡航力以上に前を取りに行くと消耗

# --- 中盤（主消耗・位置減衰）---
work_i   = L_p*pos_i + (1 - A_cruise_i)*0.5
energy_i = clip( energy_i - c_drain*work_i*(0.6 + 0.8*L_p), 0.05, 1.0)   # ハイは全体に消耗大
pos_i    = pos_i - 0.15*(pos_i - 0.5)*(1 - A_cruise_i)    # 巡航力低いと位置が中位へ流れる

# --- 終盤（決め手はエネルギーでゲート）＋終端スコア ---
w_pos_p  = w_pos0 + w_pos1*(1 - L_p)        # ★スローほど位置の重み最大＝前残りレバー
kick_i   = A_finish_i * energy_i            # 余力が無いと決め手は出ない
S_i,p    = A_class_i * ( w_pos_p*pos_i + kick_i )      # 終端スコア（A_class は変調乗数）
# --- 展開項（v4.1）: 合成器著作 leg_advantage を脚質ぶん乗算変調 ---
S_i,p   *= clip(1 + LEG_ADV_GAIN * leg_adv_p[style_i]/2, 0.2, 2.0)   # pace_level では出ないコース固有の前残り/差し台頭の残差を率に反映
```

直観：
- **スローで前**（pos高×L低）→ 消耗ほぼ無し＆位置リードを保持して直線へ → **前残り**（逃げ・先行が残す）。
- **ハイ/競り合い**（pos高×L高）→ 前が消耗して終盤に飲まれる → **差し台頭**。
- **A_cruise が高い馬**は位置を安く保ち余力を温存（道中安定）。
- 同一パラメータで pace_level だけ変えれば前残り↔差し台頭が**創発**（脚質別有利不利を手で固定しない）。

### 7. パターン別 softmax（条件付き勝率）

各パターン p で終端スコア `S_·,p` を softmax で勝率シェアに変換：
```
p_i,p = exp(S_i,p / T) / Σ_j exp(S_j,p / T)        # = パターン p の条件付き勝率
```
`L_p`（pace_level）は pace-synthesis.md のパターン・ラベルから決める：
```
L_p = clip( base_by_label + 0.15*(先行争い当事者数 - 1), 0, 1)
  超スロー 0.1 / スロー 0.25 / 平均 0.5 / やや速 0.65 / ハイ 0.85
```

### 8. パターン加重で最終勝率

```
p_i = Σ_p prob_p * p_i,p
```
`p_i` が「自分の勝率」%。**各パターンの条件付き勝率 `p_i,p` は潰さず保持**しレポート・ログに併記（§3・§4 of output-template）。

### 9. 着順分布（複勝率・連対率・予想着順）（v3.0 から不変）

Harville 近似で各パターンの勝率 `p_i,p` から着順分布を作り、パターン加重する。
```
P(iが1着) = p_i
P(iが2着) = Σ_p prob_p * Σ_{k≠i} p_k,p * p_i,p/(1 - p_k,p)
P(iが3着) = Σ_p prob_p * Σ_{k≠i}Σ_{l≠i,k} p_k,p * p_l,p/(1-p_k,p) * p_i,p/(1-p_k,p-p_l,p)
複勝率 p3_i = P(1着)+P(2着)+P(3着)
```
**予想着順**: `tools/score_race.py` は期待着順を Bradley-Terry 対比近似 `E[rank]_i = 1 + Σ_{j≠i} p_j/(p_i+p_j)` で出す（複勝率降順と整合）。

> **当日可変（SKILL STEP6）**: 当日情報でパターン確率 `prob_p` を付け替える／1パターンに固定した場合、
> §7-9 を**再計算するだけ**（§2-6 の相別能力は再調査不要）。`score_race.py` に新しい `prob`/`pace_level` を渡して再実行する。

### 10. 偶然（信頼区間）（v3.0 から不変）

点推定 `p_i` のブレ幅。主因は**パターン間のばらつき**（展開待ちの馬ほど広い）：
```
σ_i ≈ std_p(p_i,p) + p_i * (0.10 + 0.10*低確信度割合)
勝率レンジ = [p_i - σ_i, p_i + σ_i]（0未満は0でクリップ）
```

### 展開成果物の逆生成（検証用・整合チェック）

`score_race.py` は各パターンについて**派生** `leg_advantage`（脚質群平均 S − 全体平均 S を −2..+2 符号化）と
`formation_head`（序盤後 pos 上位3）・`formation_last_corner`（中盤後 pos 上位）を返す。
これは pace-synthesis.md が**著作した** leg_advantage/formation（検証軸の正本）との**整合チェック用**であり、
食い違えば STEP4a で解消する（派生を二重の正本にしない）。

## パラメータ早見表（調整対象＝この7ノブのみ）

| 記号 | 意味 | 初期値 |
|------|------|--------|
| `k_pos` | 序盤の位置取り広がり | 1.6 |
| `c_drain` | 中盤の基本消耗 | 0.45 |
| `g_fwd` | 前を取りに行く消耗結合 | 0.35 |
| `w_pos0` | 終端の位置重み（ハイ時 L=1） | 0.8 |
| **`w_pos1`** | スロー化で増える位置重み（**バイアス・レバー**、L=0で計3.4） | 2.6 |
| **`T`** | softmax 温度（S スケール 0..~2 上） | 0.75 |
| **`LEG_ADV_GAIN`** | 合成器 leg_advantage の終端S変調ゲイン（**展開項レバー**＝v4.1。コース固有の前残り/差し台頭の残差を率に効かせる。0=従来無効） | 0.20 |

**構造定数（非tunable）**: `CLASS_FLOOR=0.60`（A_class 圧縮帯）、中盤消耗係数 `0.6/0.8`、位置減衰 `0.15`、
序盤ペース抑制 `0.5`、energy 下限 `0.05`、スクレイパ・ルックアップ（ten/sty）。これらはノブ数を6に固定するため動かさない。

> 着順精度の主役は **§6 の `w_pos1`（伝達ゲイン）と `T`（較正）**。review はまずこの2つを動かす。
> **市場（オッズ・人気・α）は存在しない。** モデルは馬の内在情報のみで完結する。
> review は「同型ミスが ≥3レース 再発」して初めて1ノブずつ小刻みに動かす（単発では動かさない）。
> **ノブ変更の運用規律（v4.3〜）**: 変更前に `tools/backtest.py --set` で過去コーパスA/B → 採用したら
> `tools/record_change.py add` で**台帳記録（1変更=1記録・複数ノブ同時変更禁止）** → 導入後≥10レースで
> `compare`（反実仮想A/B）により効果測定。26件規模のコーパスへの多ノブ同時最適化＝過学習なので禁止。
