# 調査プロトコル（各観点サブエージェント用）

各観点サブエージェントは、この手順に従って web 調査を行い、末尾の**出力スキーマ**で返す。

## 共通ルール

- **対象は全出走馬**。漏れなく1頭ずつ評価する。馬番・馬名を必ず明記。
- **出典 URL を必ず残す**。推定・憶測は「推定」と明記し確信度を下げる。
- 数値（タイム・指数・オッズ・馬体重）は具体値で。曖昧なら範囲か「不明」。
- 取得できなかった項目は捏造しない。欠損として注記し確信度を下げる。
- **評価スコアは -2..+2 の5段階**（この観点単独でのその馬への寄与）：
  - +2 強い好材料 / +1 やや好材料 / 0 中立・平均的 / -1 やや懸念 / -2 強い懸念
  - 観点 I（リスク）は 0..-2 中心（減点専用）。観点 J（市場）は後述の特則。

## 推奨ソース

| 種別 | サイト/媒体 |
|------|-------------|
| 総合・出走表・成績・オッズ | netkeiba（db.netkeiba.com / race.netkeiba.com） |
| 公式・確定情報 | JRA 公式（jra.go.jp） |
| 予想・指数・展開 | 競馬ラボ、ウマニティ、各スポーツ紙（スポニチ/デイリー/サンスポ 競馬面） |
| 調教 | netkeiba 調教欄、スポーツ紙の追い切り評価 |
| 血統 | netkeiba 血統ページ、JBIS |

検索は「馬名 + 観点キーワード」「レース名 + 年 + 出走表/オッズ/追い切り」等で。
サイトを絞ってよいが、1観点で最低2ソースの突き合わせを推奨。

## 観点別の手順・検索クエリ例・抽出項目

### A. 能力指数・時計
- クエリ: `<馬名> タイム指数`, `<レース名> <年> レースレベル`, `<馬名> 上がり3F`
- 抽出: 各馬のベスト指数/平均指数、好時計を出したレースのレベル、上がり最速実績。
- 確信度: 指数の出典が複数一致なら高、単一推定なら中〜低。

### B. 近走内容・クラス通用度
- クエリ: `<馬名> 近走 着差 敗因`, `<馬名> 昇級`, `<レース名> 過去 レースぶり`
- 抽出: 直近3〜5走の着順/着差/展開、敗因分類、昇降級での通用、底を見せたか。

### C. 血統
- クエリ: `<馬名> 血統`, `<父名> 産駒 <距離/馬場> 成績`, `<母父名> 産駒 傾向`
- 抽出: 父/母父の得意距離・馬場・成長型、母系の活力、当該条件との相性。

### D. コース/距離/馬場適性
- クエリ: `<馬名> <競馬場> 実績`, `<馬名> <距離>m 成績`, `<馬名> 道悪 重馬場`
- 抽出: 当該コース/距離/馬場の連対率・着順、コース替わり/距離替わりの方向性。

### E. 展開・脚質 × 枠・ペース 〔場全体〕
- クエリ: `<レース名> <年> 展開 ペース 予想`, `<競馬場> <距離> 枠順 有利`, `<競馬場> 馬場傾向`, `<競馬場> <距離> コース 坂 形状`
- 抽出: 各馬の脚質（逃/先/差/追）、テン速、想定先行争い、想定ペース（H/M/S・前半通過目安）、トラックバイアス、枠の有利不利、坂の位置。
- **まず場全体の展開図を描いてから**各馬がその展開で恵まれるか不利かを評価。
- **必ず構造化して返す**（`観点点メモ`に格納。レース後の A/B/C 仕分け照合材料になる）:
  1. 脚質分類表（全馬 / 逃先差追 + テン速 + 想定位置）
  2. 隊列推移3フェーズ（序盤・中盤・終盤、先頭→後方を馬番で。1行隊列図可）
  3. 想定ペース（H/M/S + 前半通過目安 + ハナ争いの鍵馬 + 坂/馬場がどのフェーズで効くか）
  4. 反証条件（想定が崩れる条件を1行）
- 各馬の評価は「この想定展開での有利/不利」。前提展開依存の強い馬は前提を注記。

### F. 調教・厩舎仕上げ
- クエリ: `<馬名> 追い切り 評価`, `<馬名> 調教 時計`, `<厩舎名> 最近 成績 勝負`
- 抽出: 最終追いの時計/動き/併せ結果、調教場所・本数、厩舎の勝負気配。

### G. ローテ・馬体・間隔
- クエリ: `<馬名> 前走 いつ`, `<馬名> 馬体重 推移`, `<馬名> 休み明け`
- 抽出: 中何週/連闘/休み明け、叩き何走目、距離延長短縮、馬体重トレンド、輸送有無。

### H. 当日気配・パドック・直前情報
- クエリ: `<レース名> <日付> パドック`, `<馬名> 返し馬`, `<レース名> 直前 オッズ 動き`
- 抽出: パドック/返し馬評価、発汗・気配、直前のオッズ異変、関係者コメント。
- **取得できなければ無理に埋めず空欄＋確信度「低」**。レポートでユーザー補強を促す。

### I. リスク/割引要因
- クエリ: `<馬名> 出遅れ`, `<馬名> 気性`, `<馬名> 脚部不安 故障`, `<馬名> 斤量`
- 抽出: 出遅れ/掛かり癖、脚部不安、斤量増、不利枠、下降基調、初環境。
- スコアは 0（リスクなし）〜 -2（重いリスク）。

### J. オッズ・人気・市場勝率
- クエリ: `<レース名> <日付> オッズ 単勝 複勝`, `<レース名> 人気`
- 抽出: 各馬の単勝/複勝オッズ、人気順、前日→直前のオッズ変動。
- **特則**: この観点のスコアは「市場評価の強さ」を表す（人気=+、薄目=−）。
  自分の能力評価を混ぜず、市場が織り込む勝率の生データとして返す。

### K. 騎手・乗り替わり・コンビ相性
- クエリ: `<馬名> 前走 騎手`, `<騎手名> <競馬場> <距離> 成績`, `<騎手名> 今週 リーディング 調子`, `<馬名> <騎手名> コンビ 成績`
- 抽出:
  - **今走騎手と前走騎手**を突き合わせ、乗り替わり方向を分類: 強化（格上/有力騎手へ）/ 弱化（格下/主戦離脱）/ 継続（同騎手）/ テン乗り（初コンビ）。
  - 騎手の当該コース・距離の連対率/勝率、当該開催での調子（直近の勝ち星）。
  - 馬×騎手の過去コンビ成績（複数騎乗なら連対実績）。
  - 減量騎手（▲3kg/△2kg/☆1kg）の斤量利と経験面の不安。
- **スコア指針（-2..+2）**: 強化乗り替わり＋好相性＋コース得意 = +2 寄り／継続・中立 = 0／弱化乗り替わり・コース不得手・難しい馬へのテン乗り = -2 寄り。
- 確信度: 前走騎手・今走騎手が確定し騎手成績が複数ソースで取れれば高。出走表が暫定なら中〜低。
- **データ欠損時**: 前走騎手が取得できなければ乗り替わり判定は保留（継続=0 とみなさず確信度を下げる）。捏造しない。

## 出力スキーマ（メインに返す形式）

```
観点: <ID. 名称>
確信度(観点全体): 高 | 中 | 低
場全体メモ: <Eは上記4点（脚質分類表/隊列推移3フェーズ/想定ペース/反証条件）を必ず構造化。Jならオッズ表など観点横断のメモ。なければ空>
馬ごと:
  - <馬番> <馬名>:
      好材料: ["…", "…"]
      懸念:   ["…", "…"]
      評価:   <-2..+2>      # 観点Iは0..-2 / 観点Jは市場強度
      確信度: 高 | 中 | 低
      出典:   [<URL>, …]
注記: <データ欠損・推定箇所・ユーザー補強が望ましい点>
```

調査結果の全文は `data/races/<race-id>/research-<観点ID>.md` に保存し、
上記スキーマの構造化要約をメインへ返すこと。
