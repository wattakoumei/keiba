# 調査プロトコル（各観点サブエージェント用）

各観点サブエージェントは、この手順に従って web 調査を行い、末尾の**出力スキーマ**で返す。

## 共通ルール

- **対象は全出走馬**。漏れなく1頭ずつ評価する。馬番・馬名を必ず明記。
- **出典 URL を必ず残す**。推定・憶測は「推定」と明記し確信度を下げる。
- 数値（タイム・指数・馬体重）は具体値で。曖昧なら範囲か「不明」。
- 取得できなかった項目は捏造しない。欠損として注記し確信度を下げる。
- **評価スコアは -2..+2 の5段階**（この観点単独でのその馬への寄与）：
  - +2 強い好材料 / +1 やや好材料 / 0 中立・平均的 / -1 やや懸念 / -2 強い懸念
  - 観点 I（リスク）は 0..-2 中心（減点専用）。観点 E は評価を付けず証拠収集に徹する（後述）。
- **純粋情報の原則（v3.0）**: オッズ・人気・オッズ変動、他人の予想・印・予想展開記事は**証拠にしない**。
  関係者コメント（戦法・自信度）と媒体の主観評価（調教評価・パドック点数）は純粋情報として採用してよい。

## 推奨ソース

| 種別 | サイト/媒体 |
|------|-------------|
| **脚質・血統・枠・性齢・斤量・騎手・当日カード（核データ）** | **`tools/fetch_racecard.py`（JRA公式優先→競馬ラボ。純粋情報・観点E/C/B の主シード。`references/scraping.md`）** |
| 総合・出走表・成績 | netkeiba（db.netkeiba.com / race.netkeiba.com） |
| 公式・確定情報 | JRA 公式（jra.go.jp） |
| 調教 | netkeiba 調教欄、スポーツ紙の追い切り評価 |
| パドック・気配・コメント | スポーツ紙の馬体・パドック寸評、関係者/騎手コメント |
| コース傾向・馬場 | netkeiba コースデータ、開催の馬場傾向記事（**事実部分のみ**） |
| 血統 | スクレイパ seed（父母）＋ netkeiba 血統ページ、JBIS で補強 |

> **使わないソース**: オッズ/人気ページ、予想家・専門紙の「印」や予想展開そのもの。
> コース傾向・馬場記事は「事実（前残り傾向・内外バイアス等）」を採り、他人の本命/結論は採らない。
検索は「馬名 + 観点キーワード」「レース名 + 年 + 出走表/追い切り」等で。1観点で最低2ソースの突き合わせを推奨。

## 観点別の手順・検索クエリ例・抽出項目

### A. 能力指数・時計
- クエリ: `<馬名> タイム指数`, `<レース名> <年> レースレベル`, `<馬名> 上がり3F`
- 抽出: 各馬のベスト指数/平均指数、好時計を出したレースのレベル、上がり最速実績。
- 確信度: 指数の出典が複数一致なら高、単一推定なら中〜低。

### B. 近走内容・クラス通用度
- クエリ: `<馬名> 近走 着差 敗因`, `<馬名> 昇級`, `<レース名> 過去 レースぶり`
- 抽出: 直近3〜5走の着順/着差/展開、敗因分類、昇降級での通用、底を見せたか。
- **近走の実際の脚質・位置取り（通過順）も記録**（STEP4a 展開合成が E の脚質読みと突き合わせる素材）。
  `fetch_racecard.py` の JRA経路なら**前走通過順が seed に入る**。近走JRA成績が無い馬（海外組等）のみ web 調査で補う。

### C. 血統
- **父・母はスクレイパ seed で取得済み**（`fetch_racecard.py`）。ゼロから引かず、seed を起点に得意距離/馬場/成長型を調べる。
- クエリ: `<父名> 産駒 <距離/馬場> 成績`, `<母父名> 産駒 傾向`
- 抽出: 父/母父の得意距離・馬場・成長型、母系の活力、当該条件との相性。seed の父母名が web と食い違えば確信度を下げ web を優先。

### D. コース/距離/馬場適性
- クエリ: `<馬名> <競馬場> 実績`, `<馬名> <距離>m 成績`, `<馬名> 道悪 重馬場`
- 抽出: 当該コース/距離/馬場の連対率・着順、コース替わり/距離替わりの方向性。
- 当該コースの**前残り傾向・内外バイアスの事実**も記録（STEP4a の素材）。

### E. 展開証拠の収集 〔生証拠のみ・評価を付けない〕
- クエリ: `<競馬場> <距離> 枠順 有利`, `<競馬場> 馬場傾向`, `<競馬場> <距離> コース 坂 形状`, `<馬名> 脚質 通過順`
- **この観点は確率もパターンも有利不利スコアも出さない。** 展開合成（STEP4a）の素になる**事実だけ**を集める。
- **脚質分類はスクレイパ seed を起点に検証・補強**する（`fetch_racecard.py`）。JRA経路なら**通過順由来の精密脚質・テン速**が seed に入る（信頼度高）。競馬ラボ経路は粗い傾向バーなので近走で裏取りする。想定位置を足す。
- **必ず構造化して返す**（`場全体メモ`に格納）:
  1. 脚質分類表（全馬 / 逃先差追 + テン速 + 想定位置）
  2. 先行争いの当事者（ハナを主張しうる馬・引きそうな馬を名指し＝ペースの分岐点）
  3. 馬場バイアス（内/外・前/差し、馬番名指し。坂形状・どのフェーズで効くか）
  4. 枠の有利不利（枠順から見た位置取りのしやすさ）
- ※ 確率付きパターン・隊列・脚質別有利不利・反証条件は **STEP4a が作る**（pace-synthesis.md）。他人の予想展開記事は使わない。

### F. 調教・厩舎仕上げ
- クエリ: `<馬名> 追い切り 評価`, `<馬名> 調教 時計`, `<厩舎名> 最近 成績 勝負`
- 抽出: 最終追いの時計/動き/併せ結果、調教場所・本数、厩舎の勝負気配。
- 媒体の調教評価（S評価等）も観察情報として採用可（時計・動きの事実を優先）。

### G. ローテ・馬体・間隔
- クエリ: `<馬名> 前走 いつ`, `<馬名> 馬体重 推移`, `<馬名> 休み明け`
- 抽出: 中何週/連闘/休み明け、叩き何走目、距離延長短縮、馬体重トレンド、輸送有無。

### H. 当日気配・パドック・直前情報
- クエリ: `<レース名> <日付> パドック`, `<馬名> 返し馬`, `<馬名> 陣営 コメント`
- 抽出: パドック/返し馬評価、発汗・気配、**関係者・騎手のコメント（戦法・出方・自信度・勝負気配）**、媒体のパドック点数。
- **直前オッズ・人気・オッズ変動は抽出しない（市場情報のため排除）。**
- **取得できなければ無理に埋めず空欄＋確信度「低」**。レポートでユーザー補強を促す。

### I. リスク/割引要因
- クエリ: `<馬名> 出遅れ`, `<馬名> 気性`, `<馬名> 脚部不安 故障`, `<馬名> 斤量`
- 抽出: 出遅れ/掛かり癖、脚部不安、斤量増、不利枠、下降基調、初環境。
- スコアは 0（リスクなし）〜 -2（重いリスク）。

### K. 騎手・乗り替わり・コンビ相性
- クエリ: `<馬名> 前走 騎手`, `<騎手名> <競馬場> <距離> 成績`, `<騎手名> 今週 リーディング 調子`, `<馬名> <騎手名> コンビ 成績`
- 抽出:
  - **今走騎手と前走騎手**を突き合わせ、乗り替わり方向を分類: 強化（格上/有力騎手へ）/ 弱化（格下/主戦離脱）/ 継続（同騎手）/ テン乗り（初コンビ）。
  - 騎手の当該コース・距離の連対率/勝率、当該開催での調子（直近の勝ち星）。
  - 馬×騎手の過去コンビ成績（複数騎乗なら連対実績）。
  - 減量騎手（▲3kg/△2kg/☆1kg）の斤量利と経験面の不安。
  - **騎手の出方（行く/控える）** を記録（STEP4a 展開合成の素材）。
- **スコア指針（-2..+2）**: 強化乗り替わり＋好相性＋コース得意 = +2 寄り／継続・中立 = 0／弱化乗り替わり・コース不得手・難しい馬へのテン乗り = -2 寄り。
- 確信度: 前走騎手・今走騎手が確定し騎手成績が複数ソースで取れれば高。出走表が暫定なら中〜低。
- **データ欠損時**: 前走騎手が取得できなければ乗り替わり判定は保留（継続=0 とみなさず確信度を下げる）。捏造しない。

## 出力スキーマ（メインに返す形式）

```
観点: <ID. 名称>
確信度(観点全体): 高 | 中 | 低
場全体メモ: <E は下記4点（脚質分類表/先行争い当事者/馬場バイアス/枠）を構造化。他観点で場全体の所見があれば。なければ空>
馬ごと:
  - <馬番> <馬名>:
      好材料: ["…", "…"]
      懸念:   ["…", "…"]
      評価:   <-2..+2>      # 観点Iは0..-2 / 観点Eは評価を付けず証拠のみ
      確信度: 高 | 中 | 低
      出典:   [<URL>, …]
注記: <データ欠損・推定箇所・ユーザー補強が望ましい点>
```

### 観点E の証拠スキーマ（PACE_EVIDENCE_SCHEMA）
E は上記の馬別評価ではなく、次の生証拠を返す（STEP4a が消費する）：

```json
{
  "legs": [{"no": 4, "name": "〇〇", "style": "逃げ", "ten_speed": "速", "expected_pos": "単騎ハナ"}],
  "lead_contenders": [{"no": 9, "stance": "ハナ主張しうる"}, {"no": 4, "stance": "引かない"}],
  "bias": {"track": "前/内有利", "detail": "坂は最後の1F、外差しは届きにくい", "key_horses": [4, 2]},
  "draw": [{"no": 1, "note": "最内・包まれ懸念"}]
}
```

調査結果の全文は `data/races/<race-id>/research-<観点ID>.md` に保存し、
上記スキーマの構造化要約をメインへ返すこと。
