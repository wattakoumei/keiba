# review-prediction — 予測精度の検証と改善ループ（2スコアカード）

`/analyze-race` の予測（`data/predictions.jsonl`）と実結果を突合し、
`scoring-model.md`・`pace-synthesis.md` の改善案を提示する。改善ループの 1 周を担う。

> **北極星（v5.0・論理ファースト）**: 検証対象は **展開予想と着順予想の精度**であって回収率ではない。馬券・市場は持たない。
> **2つの独立スコアカード**で測る（予測側に % が無いので、採点も**質的・順位ベース**が主）：
> - **スコアカード1 展開精度**: 実現パターンの的中（`likelihood_tier` 最上位が実現したか）・脚質別有利不利の符号・隊列(`formation_*`)・**段階フロー `phase_flow`**（序盤の隊列→中盤の消耗→終盤の台頭）が実通過順と合うか（ラップ非依存）。
> - **スコアカード2 着順精度**: **並びの順位相関**（予測 `rank_order`／印 vs 実着順の Spearman/Kendall）・◎単勝/上位複勝的中・**好材料/懸念点の的中**（読んだ材料が実際に効いた/外したか）。
>
> 市場（オッズ）は使わないので **モデル vs 市場の比較・α は無い**。
> **Brier・キャリブレーション**は予測側に %（`win_prob`）が無くなったため**第一指標から降格**（v4.0 以前の数値ログがある旧レースでのみ参考算出可）。
モデル定義は repo ルート `CLAUDE.md`、調整点は `pace-synthesis.md`（展開合成手順・段階フロー）と `observation-points.md`（相別マッピング・読み筋）。`scoring-model.md` の 6ノブはエンジン併用時のみの任意調整。

## 手順

### STEP 1. 結果の記録

対象レースの確定着順を web 取得 or ユーザー貼り付けで集め、`data/results.jsonl` に追記。
**展開の復元に使う通過順・上がりも取れる範囲で記録**する。1レース1行：

```json
{"race_id":"20260531-tokyo-11","date":"2026-05-31","finish":[{"no":5,"name":"〇〇","pos":1,"passing":"3-3","agari":34.5,"style":"差し"},{"no":4,"name":"△△","pos":2,"passing":"1-1","agari":35.2,"style":"逃げ"}],"pace_actual":{"first_600m":34.8,"label":null,"label_reconstructed":"S","reconstructed_from":["勝ち先行②残存","3-4角先頭が確保","上がり最速馬は着外"]}}
```

- `pace_actual.label`（実ラップのH/M/S）が取れなければ `null`。
- **`label_reconstructed`**: ラップが無くても (a)上位馬の脚質（前残りか差し決着か）、(b)3-4角通過順と着順の相関、
  (c)上がり最速馬の着順 から実効ペース層(H/M/S)を**復元**する。`reconstructed_from` に根拠を残す。

### STEP 2. 展開スコアカード（成果物1の採点）— 先に行う

`report.md §2` / `predictions.jsonl` の `record:"pace"` を実結果と照合：

- **実現パターンの特定**: `patterns[].trigger` と実レース（誰がハナを取ったか・隊列）を照合し、どのパターンが起きたかを決める。
- **パターン的中**: **本線（`likelihood_tier`最上位）パターン**が実現パターンと一致したか。＋実現パターンにどのティアを置けていたか。
- **脚質別有利不利の的中（主指標・ラップ不要）**: 予測 `leg_advantage` の符号 vs 実際に好走した脚質層（上位3頭の脚質構成）。
- **隊列的中**: `formation_head` vs 実際の3-4角先頭、`formation_last_corner` vs 実通過順。
- **段階フローの的中（v5.0）**: 実現パターンの `phase_flow`（序盤の隊列→中盤の消耗→終盤の台頭）が実通過順・上がりの推移と合うか。
  ＝「どの相で誰が浮く/沈むと読んだか」が当たったか。これが §3 展開感度の検証であり、A/B 切り分けの核。
- **バイアス的中**: 内外/前後予測 vs 実決着傾向。

`results.jsonl` に展開採点を追記（`record:"pace_review"`）：
```json
{"record":"pace_review","race_id":"20260531-tokyo-11","realized_pattern":"α","pattern_hit":true,"leg_advantage_hit":{"逃げ":true,"先行":true,"差し":true},"formation_head_hit":true,"phase_flow_hit":true,"bias_hit":true}
```

### STEP 2.5. A/B/C 自動一次仕分け（展開採点を起点に）

展開スコアカードの結果で、予測した上位馬の外し方を機械的に一次仕分けする（人手は C 認定と境界事例に集中）：

| 種別 | 自動判定（v5.0） | 反映先 |
|------|---------|--------|
| **A 能力読み違い** | **展開的中**（実現パターン＝本線、脚質有利不利＋`phase_flow` hit）なのに**並び（印・rank_order）が実着順とずれた**。＝展開は合ったが能力の高低・展開感度を読み違えた | **`observation-points.md` 相別マッピング・好材料/懸念の読み筋**（A_early/cruise/finish の配分）。エンジン併用時のみ補助で 6ノブ `w_pos1`/`T`/`CLASS_FLOOR` |
| **B 展開読み違い** | 展開を外した（実現パターン≠本線 or 脚質有利不利が逆 or `phase_flow` の段階推移が違った） | **`pace-synthesis.md`・`pace_level`/`likelihood_tier`・観点E手順（最重要）** |
| **C 偶然** | 落馬・致命的不利・故障・大出遅れ等、事前に織り込めない事象（人手認定） | **触らない。母数から除外** |

> **v5.0 の A/B 切り分け**: 実現パターンの **`phase_flow`・脚質有利不利が当たれば「展開は読めた」**。その上で並び（印）が実着順とずれたら **能力読み＝A**、
> phase_flow/脚質有利不利から外していたら **展開読み＝B**。
> **加古川特別が A の代表例**: 前残りを的中させつつ追込◎を上位に置いた＝段階フローは読めたが終盤の決め手（A_finish）と位置（A_early）の重み付けを誤った。
> 修正先は好材料/懸念点の読み筋＝相別マッピング（A_early と A_finish を別物として扱えていたか）。review は「同型ミスが ≥3レース 再発」して初めて読み筋を見直す（単発では動かさない）。

`miss_class` を追記（実データ mizusawa-10 は「展開的中×能力外し」で 5/5=A になる）：
```json
{"race_id":"20260531-tokyo-11","miss_class":[{"no":10,"class":"A","note":"展開(前残り)的中、序盤の位置取り(A_early)を過小評価し並びを下げ過ぎ"}]}
```

### STEP 3. 着順スコアカード（成果物2の採点）

`predictions.jsonl` の `record:"rank"` と実着順を突合（**C は母数から除外**）：

- **並びの順位相関（最重要）**: 予測の **`rank_order`**（印の強さ順）と実着順の **Spearman ρ / Kendall τ**。予測側に % は無いので、相関は**順位ベース**で取る。
- **的中率**: ①◎単勝的中率（本命1着）②印持ち上位3頭（◎◯▲）の複勝的中率。
- **好材料/懸念点の的中（v5.0・質的）**: 各馬の `pros[]`/`cons[]` を実走と照合 ―「[E]好枠で前残り恩恵」と読んだ馬が実際に前で残ったか、「[I]折り合い懸念」が実際に出たか。
  読んだ材料が**効いた/外した/無関係**を質的に判定し、観点別に集計（どの観点の読み筋が当たりやすい/外しやすいか）。
- **★純粋な能力読みの分離計測**: **実現パターンの `phase_flow` と展開感度（`pace_sensitivity`）**が当たったかを別途見る。
  展開（phase_flow）が当たったのに並びがずれれば「能力の高低を読み違えた」＝A、phase_flow から外していれば「展開を外した」＝B。**A/B 切り分けの裏付け**になる。
- **（旧レース参考）**: v4.0 以前の行に `win_prob`/`conditional[].win_prob` が残っていれば Brier も参考算出してよい（新行には無い＝第一指標にしない）。

> 少数レースでは荒れる。サンプル数を必ず併記し、1〜2レースで読み筋を大きく動かさない。

### STEP 4. 観点バイアス診断

各観点について、好材料/懸念点で高/低に読んだ馬の実着順傾向から偏りを探す（C 除外母数で）。例：
- 「F 調教を好材料に挙げた馬の着順が伸びていない → 調教の読みが過大」
- 「I で重く懸念した馬が好走（mizusawa ⑩のテン乗り割引）→ リスクの読みが過大」
- 「E 由来の展開感度で恵まれると見た馬が着外続き → `pace_level`/`phase_flow` or 展開合成の精度」
- 「前残り想定で前が残らない／差し想定で差しが届かない → `phase_flow` の段階読み or `pace_level` 写像」

観点別に **過大 / 過小 / 妥当** を判定し、根拠（どのレース・どの馬）を添える。
**B が複数レースで再発**（例: スロー想定→実ハイが繰り返す）したら `pace_level` 写像・`phase_flow`・観点E手順・pace-synthesis.md を見直す根拠に。単発の B は動かさない。

### STEP 5. 修正案の提示（v5.0・読み筋ファースト）

診断に基づき**具体的な読み筋／手順の修正案**を、スコアカード別にフィードバック先を分けて出す：
- **展開スコアカード発（B 系）** → `pace-synthesis.md` の合成・パターン分岐・`pace_level`/`likelihood_tier`/`phase_flow` の段階読み、`observation-points.md` の観点E手順。
- **着順スコアカード発（A 系）** → `observation-points.md` の**相別マッピング**（A_early と A_finish を別物として扱えていたか）と**好材料/懸念点の読み筋**。
- 「序盤の速さ（A_early）を逃げ馬で過小評価→前残りで取りこぼし。脚質→A_early の読みを強める」のように**読み筋の変更・理由・期待効果**をセットで。
- **エンジン併用時のみ（任意）**: 同型ミスが ≥3レース再発したら `scoring-model.md` の 6ノブ（主に `w_pos1`/`T`）を 1個ずつ小刻みに。変えたら `tools/score_race.py` の `PARAMS` を**必ず同値にミラー**する。

### STEP 6. 承認 → 更新（バージョンを上げる）

ユーザー承認を得たら該当ファイルを編集する。`pace-synthesis.md`/`observation-points.md` の読み筋を変えた場合はその旨を記す。
エンジン（6ノブ）を変えた場合のみ `scoring-model.md` のモデルバージョンを上げ「変更履歴」に追記。これでループ 1 周完了。

### STEP 7. 回帰確認（任意）

可能なら同じ過去レースを更新後の読み筋で再分析し、**展開精度（脚質有利不利・段階フローの的中）と着順精度（並びの順位相関・好材料/懸念の的中）が
別々に**改善方向かを目視確認。複数レースが溜まっていれば通算で2スコアカードを再集計する。

## 注意

- サンプルが少ないうちは過剰反応しない。検証は結果論の印後付けでなく、2スコアカードの客観指標（順位相関・段階フロー的中・材料的中）で行う。
- **後方互換**: 旧行（`record` 無し・v1.x/v2.0 の `my_prob`/`edge`/`ev_win`、v3.0/v4.0 の `win_prob`/`phase_abilities`/`conditional[].win_prob`）はそのまま読める。
  v5.0 新行は %見出しを持たず `mark`/`rank_order`/`pace_sensitivity`/`pros[]`/`cons[]` を持つ。共通キー `leg_advantage`/`formation_*` は全世代で同じに読める。
  順位相関は v5.0 行は `rank_order`、旧行は `predicted_rank`/`win_prob` 降順から取る。Brier は %を持つ旧行でのみ参考算出。
- 読み筋は `pace-synthesis.md`（展開・段階フロー）と `observation-points.md`（相別マッピング・好材料/懸念）で管理。エンジンの 6ノブは `scoring-model.md`＝`score_race.py` の `PARAMS`（任意・同値ミラー）。複数箇所に散らさない。
