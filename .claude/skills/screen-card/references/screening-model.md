# 選別モデル（勝負レース選別・正本）

> **役割**: `/screen-card`（勝負レース選別ハーネス）の**モデルと出力契約の正本**。
> 指定した**日付＋開催場**のカードから「妙味のあるレース＝勝負レース」を、予想本体(`/analyze-race`)とは**別レイヤー**で絞る。
> **🚧 I1-S 隔離**（[[harness-invariants]] I1-S）: 市場（単勝オッズ）は**選別の目的に限り**読む。出力は `data/screening/` にのみ置き、research/report/predictions には流さない。`/analyze-race` はオッズを一切見ない。**馬ごとの買い目・金額・EV は出さない**（人間判断）。
> **数値閾値のワンソース**（I10）: 団子度の閾値は `tools/fetch_odds.py`（FAV_SPLIT 等）が正本。本書は**意味と向き**を述べ、数値は二重管理しない。荒れ条件は [[upset-conditions]] が正本。
> 2026-06-26 新設。

## §1. マトリクス（妙味の定義）

2軸で各レースを置く:

- **X軸＝硬さ**: うちの内在予想（市場ゼロ）で**結果を絞れるか**。主軸を1〜2頭に確信を持って置ける＝X高。
- **Y軸＝荒れ度**: **配当が大きく期待できるか**＝世間が絞れていないか。= **条件荒れ度（事前）＋ 団子度（当日オッズ）**（§2）。

|  | **Y低（収束＝堅い）** | **Y高（割れ＝荒れる）** |
|---|---|---|
| **X高（うちは絞れる）** | 🟦 **鉄板コツコツ**: 抜けた本命をうちも支持。当たるが配当安い→資金回収型 | 🟥 **本命勝負**: うちは絞れる×世間は割れてる＝当てて配当デカい。**最も妙味大** |
| **X低（うちも絞れない）** | ⬜ 妙味なし: 人気に逆らえない→見送り/軽く | ⬛ 博打: 誰も読めない→見送り or 遊び |

- **妙味の素 = うちのX主軸が市場(オッズ)で過小評価されてるズレ**。**増幅 = Y荒れ度**（割れてるほど当てた時デカい）。
- **団子≠予想不能**（B研究の核）: 団子でも「単勝30倍以下が3着内全消え」は稀（好走馬の約9割が30倍以下）。＝「圏内候補が多くて世間が1頭に絞れない」だけ。そこをうちのXで絞れれば🟥になる。

## §2. Y軸＝荒れ度の算出（2成分）

### §2a. 条件荒れ度（事前・[[upset-conditions]]）
番組情報だけで判定（オッズ不要＝**未来日付でも出せる**）。各レースに荒れ条件フラグを当て、強フラグの数で `条件荒れ度`(強/中/弱)を出す。主な強フラグ:
- **ハンデ戦**／**福島開催**／**ローカルG3ハンデ重賞**（北九州記念・愛知杯・函館記念・七夕賞 等）／**多頭数(15-18頭)** = 強。
- 芝×道悪（重/不良・**芝限定**）／芝短距離(1000-1200・ヒモ荒れ)／ローカル夏(小倉・中京・福島)／3勝クラス = 中。
- 堅い側（収束方向）: 東京・阪神/芝中長距離/新馬・未勝利/ダート/G2/中山金杯系。
- 詳細・数値根拠・確信度は [[upset-conditions]]。

### §2b. 団子度（当日・`tools/fetch_odds.py`）
単勝オッズが出ていれば取得（**発売中のみ**＝当日/前日発売中）。`fetch_odds.py` が単勝→メトリクス→ティアを返す:
- **① 1人気-2人気のオッズ差（最強）**: 小さい＝拮抗＝割れ。
- **② 1番人気の単勝オッズ**: 高い＝本命不在＝割れ／低い＝抜けた本命＝収束。
- **③ 単勝30倍以下の頭数**: 多い＝団子。 **④ 単勝平均オッズ**: 低い＝団子。
- 重み付き合計で `dango_tier`（割れ★★★／中★★／収束★）。**断層理論は不採用**（①で代替）。閾値の数値は `fetch_odds.py` が正本。
- **オッズ未公開なら団子度は欠測** → Y＝条件側のみで「事前候補」として返し、当日に再評価（§5 二段運用）。

### §2 統合
`Y荒れ度 = 条件荒れ度 × 団子度`。両方が荒れ方向＝Y高（強）。食い違い（条件は荒れ条件だが盤は収束＝抜けた本命がいる等）は中。**当日の団子度が出たら条件の事前推定を上書き**（実測優先）。

## §3. X軸＝硬さの算出（軽量X・shortlist のみ）

shortlist（🟥系=Y高の荒れ候補 ＋ 🟦系=Y低だが軸が立ちそうな堅い候補。両取り・SKILL STEP4）に**軽い読み**を入れる（全Rに深い予想はしない＝選別の主旨は「どこに /analyze-race を投下するか」）。源は `fetch_racecard.py` の seed（脚質・前走通過順・血統・近走クラス）。各候補レースで:
- **主軸候補**（1〜2頭）と、その**確信**（絞れる＝X高／割れて絞れない＝X低）。
- 根拠は内在情報のみ（**市場ゼロ**＝この段ではオッズを見ない）。1〜2行で良い。
- 必要なら候補レースごとに軽量サブエージェントを並列起動（Agentツール）。深掘りではない＝11観点ファンアウトはしない。

**市場ズレ（妙味の素）**: §3 の主軸が確定したら、§2b のオッズ順位と突き合わせる。**うちの主軸が人気薄（オッズ上位でない）ほどズレ大＝妙味大**。オッズ未公開なら「当日確認」と保留。
> ★順序の壁（I1-S）: 必ず**先にXを内在情報だけで決め**、**後で**オッズと比べる。オッズを見てから主軸を選ばない（市場汚染の防止）。

## §4. 妙味判定（X×Y → 象限ラベル）

各レースを §1 マトリクスに置く:
- X高 × Y高 → 🟥 **本命勝負**（妙味大。うちの主軸が人気薄ならさらに大）。
- X高 × Y低 → 🟦 **鉄板コツコツ**（抜けた本命にうちも乗る）。
- X低 × Y高 → ⬛ 博打（見送り推奨）。 X低 × Y低 → ⬜ 妙味なし（見送り）。
- **勝負度ランキング**: 🟥 を上に、ズレ大・荒れ度強・確信高い順。🟦 は別カテゴリで併記（回収型）。

## §5. 出力体裁（チャット）

ユーザーは表中心（結論先頭・1スクロール以内）。1カード=1表＋ランキング。

1. **カード一覧表**（**全R必ず掲載**）: 列 = `R / 条件(芝距離/クラス/種別) / 条件荒れ度 / 団子度(★or未) / X主軸・確信 / 市場ズレ / 象限`。軽量Xした候補は主軸・象限つき、評価しなかった平場は **`見送り`（条件のみ・軸未評価＝x_axis=null）** として条件荒れ度だけ付けて載せる（取りこぼし防止＝「全部見た」を示す）。条件荒れ度の機械算定は `tools/screen_conditions.py`（`assess`/`fill`）。
2. **勝負候補ランキング**: 🟥本命勝負 → 🟦鉄板コツコツ の順に、レースと一言理由。
3. **次アクション**: 勝負候補だけ「→ `/analyze-race <race>` で深掘り」。
4. **出さないもの（I1-S）**: 馬ごとの買い目・金額・点数・EV・回収率。「どのレースを・どの戦い方(本命勝負/コツコツ/見送り)で」までが選別の責務。券種と金額は人間。
5. オッズ未公開なら表の団子度・ズレ・象限は「**事前**」と明示し、当日再実行を促す（§2b/§3）。

## §6. data/screening/ スキーマ（I1-S の壁の内側）

カード集計を `data/screening/<YYYYMMDD>-<venue-romaji>.json` に保存（**ここにのみ**オッズ由来データを置く）:
```json
{
  "date": "20260628", "venue": "hakodate", "place": "02",
  "as_of": "<実行時刻>", "odds_available": true,
  "races": [
    {"r": 11, "surface": "芝", "distance": 1200, "headcount": 16,
     "class": "3勝クラス", "race_type": "ハンデ", "race_name": "…",
     "cond_rage": "強", "cond_flags": ["ハンデ","多頭数","芝短距離"],
     "dango_tier": "割れ", "dango_signals": ["…"],            // fetch_odds.py 由来。未公開なら null
     "x_axis": {"anchor": ["7 ウマ"], "confidence": "絞れる",  // 内在情報のみ
                "market_divergence": "主軸が4番人気＝ズレ大"},  // オッズ未公開なら null
     "quadrant": "本命勝負", "score": 5}
  ]
}
```
- `dango_*`・`market_divergence` は**選別専用**。**predictions.jsonl 等には絶対コピーしない**（I1-S 壁①②）。

## §7. 二段運用（事前 → 当日）

- **事前（出馬表後〜前日）**: STEP 条件荒れ度＋軽量X で「事前勝負候補」を出す（団子度・ズレは保留）。下ごしらえ＝深掘り先の目星。
- **当日（発売後）**: `fetch_odds.py race` で団子度を入れ、ズレを確定 → 象限を最終化。発売前は `paste` でオッズを貼っても可。
