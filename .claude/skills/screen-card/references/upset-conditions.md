# 荒れ条件カタログ（半静的・正本）

> **役割**: レース選別レイヤーの **Y軸（荒れ度＝配当の大きさが期待できるか）の「条件」側** の正本。
> オッズが出る前の**番組情報だけ**（種別・クラス・頭数・距離・芝ダ・コース・馬場・開催）で「このレースは荒れそう」を事前推定するための条件カタログ。
>
> **🚧 I1 ガード（最重要）**: このカタログは**選別レイヤー専用**。`/analyze-race` の**予想本体（X軸＝誰が勝つか）には一切使わない**。
> 含まれる数値は 1番人気勝率・配当・万馬券率など**市場由来**だが、それは「荒れ度＝旨味」を測る選別＝**賭ける瞬間のレイヤーにのみ隔離**して使う（市場は賭ける瞬間だけの隔離レイヤー）。予想側の市場ゼロ（harness-invariants I1）は維持される。
>
> **半静的**: 集計データは年単位で動く＝**年1回程度の更新**。`as-of 2026-06`。
> **使い方**: 選別時に対象レースの番組情報から該当フラグを抜き、加点する（合成式は §3末・未設計＝当面は人手で序列判断）。
> **確信度**: データ裏付けあり／通説寄りデータ／通説のみ／不明。**△＝母数小or単年サンプル**。
> 2026-06-26 新設（4並列 web 調査＝種別×クラス／頭数×構成／距離×芝ダ×コース／馬場×季節×重賞。出典は個人ブログ/予想サイト集計が中心＝JRA一次統計ではない。PC-KEIBA の馬場別・会場別配当が最大規模で最も信頼度高）。

## 巻頭原則（選別の一次フィルタ）

1. **「1番人気の堅さ」と「配当の荒れ」は別指標**。Y軸（旨味＝配当の大きさ）は **万馬券率／平均配当（外れ値に弱いので中央値も）** で測る。1番人気複勝率は「世間が割れてる＝うちの X軸（硬さ）が刺さる余地」の参照に回す。
2. **「荒れる」には2種類**ある。選別では区別する:
   - **超大穴型**（1着から人気薄／単勝万馬券）= 下級条件・特殊条件・ハンデ・道悪芝で発生。
   - **ヒモ荒れ型**（1着は人気だが2-3着が人気薄→3連系が高配当）= 短距離・G1 など。1着は読めても3連系で配当が膨らむ。
3. **平均配当は外れ値に弱い**（例: ヴィクトリアM=定量G1だが2015年2,070万円で平均が267万に歪む。除外時≈21万で実は堅い）。**万馬券率・1番人気複勝率・中央値**を主指標に。
4. **複合で効く**。最荒れの典型 = 福島/小倉 × 芝短距離 × ハンデ × 多頭数。最堅 = 東京/阪神 × 芝中長距離 × 定量。
5. **当初仮説の誤り（データで修正済み）**:
   - 「G1・重賞は実力差で堅い」は**誤り**。G1の1番人気複勝率52.1%が全クラス最低（伯仲＝荒れる）。ただし単勝超大穴はほぼ出ない＝「上位人気内の入替で荒れる」型。
   - 「下級ほど荒れる」も**新馬・未勝利は逆**（複勝70%前後で最堅＝調教/血統/厩舎力の情報格差で上位が圧倒）。条件戦の荒れピークは**3勝クラス**。

### 同型構成（逃げ・先行の頭数）→ 前崩れ荒れ
- 仮説「同型過多（特に短距離で逃げ4-5頭+）→ハイペース→前崩れで荒れ／逃げ不在→スロー前残りで堅い」は**通説止まり**。逃げ馬数を変数にした決着率/配当の統計は未発見。
- → **自前較正項目**（§4-1）: `tools/fetch_racecard.py` の脚質/テン速 seed から逃げ・先行頭数をカウント → `data/results.jsonl` の pace_actual・配当と突合して較正。

## 3. 運用上の注意

- **道悪荒れフラグは芝限定**（ダートに掛けると誤検出源＝§1 #5）。
- **集計期間が出典ごとにバラバラ**（2007-2023 / 2016-2026 / 2017-2021 / 2020-2025 / 2022-2024）。厳密値より**方向性・序列**を信頼する。
- **重み合成式は未設計**（§4-4）。当面は人手で「強フラグ2つ以上＝荒れ候補」程度の序列判断で運用。
