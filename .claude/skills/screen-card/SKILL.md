# screen-card — 勝負レース選別オーケストレータ

指定した**日付＋開催場**のカードを横断し、「どのレースを・どの戦い方で勝負するか」を絞る。
予想本体(`/analyze-race`)が1レースを深掘りするのに対し、これは**カード横断で投下先を選ぶ上流レイヤー**。

> **🚧 I1-S 隔離**（[[harness-invariants]] I1-S が正本）: 市場（単勝オッズ）は**選別の目的に限り**読む。
> ① オッズ由来データは `data/screening/` にのみ書く（research/report/predictions には**流さない**）。
> ② X軸（誰が勝つか）は内在情報だけで作る＝**先にXを決め、後でオッズと比べる**（順序を逆にしない）。
> ③ 出力は**レース単位の妙味判定まで**。馬ごとの買い目・金額・EV は出さない（人間判断）。

参照（このSkillの references/）:
- `screening-model.md` — **モデルと出力契約の正本**（X×Yマトリクス・Y算出・軽量X・妙味判定・出力体裁・data/screeningスキーマ）
- `upset-conditions.md` — **荒れ条件カタログ**（Y軸の条件側・正本。半静的）
- 数値閾値（団子度）は `tools/fetch_odds.py` が正本（I10）。

## 手順

### STEP 1. カード取得（全R・番組情報）
入力（日付＋開催場。例「6/28 函館」）を `YYYYMMDD` ＋ 場2桁に正規化（場コードは `references/scraping.md` の表。函館=02 等）。
- `python3 tools/fetch_racecard.py day <YYYYMMDD> <場2桁> --json` で全Rの `surface(芝/ダ)・distance・headcount・race_name・post_time` を取得。
- JRA直接で確認するなら `python3 tools/fetch_racecard.py jra <YYYYMMDD> <場2桁> --json`（今週開催のみ＝トークン一覧）。
- レース名から **クラス・種別（ハンデ/定量/重賞名）** を読む（ハンデ戦は「ハンデ」表記・重賞名は [[upset-conditions]] のリストと突合）。取れない時のみ web で補う。

### STEP 2. 条件荒れ度（Y条件側・**カードの全Rに付ける**・オッズ不要＝未来日付でも可）
**カードの全レース**に [[upset-conditions]] の荒れ条件フラグを当て、`条件荒れ度`(強/中/弱) を出す（カバレッジ＝全R）。
これは機械的（場・頭数・距離・芝ダ・クラス・重賞名から）でweb不要＝全Rやっても軽い。`tools/screen_conditions.py`（あれば）で一括算定可。
- 強: ハンデ／福島開催／ローカルG3ハンデ重賞／多頭数(15-18頭)。中: 芝×道悪／芝短距離／ローカル夏／3勝クラス。
- 開催場そのものが効く（福島・小倉=荒れ寄り／東京・阪神=堅い）。**道悪フラグは芝限定**。
- web 再調査しない（カタログがワンソース・I10）。馬場状態が分かれば芝道悪フラグに反映、不明なら良想定。

### STEP 3. 団子度（Y盤側・オッズが出ていれば）
単勝オッズが**発売中**なら取得して各レースの団子度を出す:
- `python3 tools/fetch_odds.py race <race_id12桁> --json --out data/screening/<…>.json`（JRA・今週/発売中のみ）。
- 発売前/取得失敗（exit≠0）なら **paste 経路**: JRA等のオッズ画面を人が貼り `… | python3 tools/fetch_odds.py paste <race_id> --json`（形式「馬番 単勝 [馬名]」 or JSON配列）。
- `dango_tier`(割れ★★★/中★★/収束★) と `dango_signals` を受け取る。閾値は `fetch_odds.py` が正本＝ここで再計算しない。
- **オッズ未公開なら STEP3 を飛ばし**、団子度=未取得のまま「事前候補」として進む（STEP6 で当日再実行を促す）。

### STEP 4. shortlist（🟥候補＋🟦候補を**両取り**・1パスで）
STEP2＋3 から `Y荒れ度` を決め、軽量X(STEP5)へ回す候補を **2系統まとめて** 拾う（🟥と🟦を別パスにしない＝1回の選別で両方出す）:
- **🟥系（荒れ×妙味）**: Y高＝条件荒れ度 強/中（団子割れ）のレース。
- **🟦系（堅い×回収）**: Y低でも**軸が立ちそうな堅いレース**＝少頭数・中距離(芝1800-2000)・上級/実力上位が明確そう・抜けた本命の気配があるもの。
- 両系統あわせて概ね 6〜10R に絞る（選別の主旨＝深掘り先の目星。全R深読みしない）。Y中庸で軸も立たなそうなレースは最初から外す（見送り確定）。

### STEP 5. 軽量X（硬さ＋市場ズレ・shortlist のみ）
各候補レースを**軽く**読む（`screening-model.md §3`）。源は `fetch_racecard.py race <race_id> --json` の seed（脚質・前走通過順・血統・近走クラス）。
- 主軸の**絞れ方を3段**で内在情報だけで判断（1〜2行・**オッズを見ない**）＋**脚質構成（逃/先/差/追の頭数）**:
  - **絞れる**(1頭抜け=X高) ／ **上位ボックス**(2-4頭に根拠=X中) ／ **切り**(上位すらフラット=X低)。
- **逃げ不在(逃0)は自動で『切り』**（脚質seedで判定＝展開が読めない荒れは触らない）。
- 候補が多い/重いレースは**軽量サブエージェントを並列起動**（Agentツール。1レース1体・脚質頭数も返させる）。
- 判定の出口（`screening-model.md §4`）: **絞れる×荒れ→🟥／絞れる×堅い→🟦／上位ボックス×荒れ→🎯穴ボックス／切り(逃げ不在・フラット)→✖／上位ボックス×堅い→⬜妙味なし**。勝負候補＝🟥🟦🎯。
- **その後**に STEP3 のオッズ順位と突き合わせ、`市場ズレ`（うちの主軸が人気薄ほどズレ大＝妙味大）を付す。オッズ未公開なら「当日確認」と保留。

### STEP 6. マトリクス配置・出力
各レースを X(絞れ方3段)×Y(荒れ度) で仕分け（`screening-model.md §1/§4/§5`）:
- 🟥 本命勝負（絞れる×荒れ）／🟦 鉄板コツコツ（絞れる×堅い）／🎯 穴ボックス（上位ボックス×荒れ）／✖ 切り（逃げ不在・フラット）／⬜ 妙味なし・見送り。**勝負候補＝🟥🟦🎯**。
- 出力 = **①カード一覧表**（全R・条件/荒れ度/団子度/X主軸・確信/ズレ/象限）＋ **②勝負候補ランキング**（🟥→🟦）＋ **③次アクション**（勝負候補だけ `→ /analyze-race で深掘り`）。
- **買い目・金額・点数は出さない**（I1-S③）。「どのレースを・どの戦い方で」まで。
- **screening ファイルにはカードの全Rを必ず収める**（取りこぼし防止＝「全部見た」を示す）: 軽量Xした候補は主軸・象限つき、それ以外の平場は **`見送り`（条件のみ・軸未評価＝x_axis=null）** として**条件荒れ度だけ**付けて載せる。score は候補>0 / 博打・妙味なし≈0〜-1 / 未評価平場=-3 で並べ（候補が上・平場が下）。
- カード集計を `data/screening/<YYYYMMDD>-<venue>.json` に保存（I1-S①の壁の内側）。
- **保存直後に必ず `python3 tools/validate_screening.py <保存したファイル>` を回して 0 で終わることを確認**（スキーマドリフト＝x_axis文字列・絵文字象限・独自キーは keiba-web の CI ビルドを落とす。当日更新で既存ファイルに追記した時も同様）。
- オッズ未公開だった場合は表に「**事前**」と明示し、当日（発売後）に STEP3 から再実行するよう促す（二段運用 `screening-model.md §7`）。

## NAR 差分（地方競馬のカードを絞る）

場トークン（`ooi`/`sonoda` 等＝`analyze-race-nar` 差分1の一覧・`inject_probs.py NAR_VENUES` とミラー）で起動（例「7/9 大井で勝負レースを絞って」）。手順は STEP1-6 のまま、以下だけ差し替え:

- **STEP1**: `fetch_racecard.py` は JRA 専用＝使わない。keiba.go.jp 公式の当日開催カードを web で読み、**カードJSON** `[{r, surface:"ダ", distance, headcount, race_name, class, post_time}]` を scratchpad に整形（`class` はクラス表記をそのまま。混合戦は全表記＝「C1二 C2一」等・これが混合判定の素）。
- **STEP2**: `python3 tools/screen_conditions.py assess <YYYYMMDD> <venue> --card <path|-> [--json]`／`fill <YYYYMMDD> <venue> --card <path|->`。場トークンで NAR 自動分岐→ [[upset-conditions]] §5（NAR差分カタログ・暫定）を適用（混合戦・ハンデ・多頭数=荒れ／少頭数・交流Jpn・フレッシュ=堅い。**道悪フラグは使わない**＝ダートは効かない）。
- **STEP3**: `fetch_odds.py race`（JRA経路）不可＝**`paste` 経路のみ**（楽天競馬・オッズパーク・keiba.go.jp のオッズ画面を人が貼る）。少頭数では③30倍以下頭数④平均が効きにくい＝**①1人気オッズ②1-2人気差が主**。
- **STEP5 軽量X**: fetch_racecard の seed が無い＝カードの**所属・クラス表記から観点N的読み**（混合戦の格上・転入格上）を第一軸に、必要なら軽量サブエージェントで補う。**転入格上の初戦**は「Xは立つが初戦割引（nar-class-ladder 換算原則2）」を確信度に反映。脚質が取れなければ**逃げ不在の自動切りは適用しない**（上位ボックス判定に切替）。
- 保存は同スキーマ `data/screening/<YYYYMMDD>-<venue>.json`（`place`=場トークン・`nar:true`）。I1-S の壁は同一。保存直後の `validate_screening.py` ゲートも同一。

## メモ
- **二段運用**: 事前（出馬表後）=条件＋軽量Xで目星 → 当日（発売後）=団子度・ズレを入れて確定。
- **速度**: カード横断なので軽さ優先。深い予想は選ばれたレースの `/analyze-race` に委ねる＝ここで作り込まない。
- **市場の壁を破らない**: 困ったら [[harness-invariants]] I1-S に戻る。オッズは選別の箱の外（research/report/predictions）に絶対出さない。
