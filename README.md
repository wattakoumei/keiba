# keiba — 競馬予想ハーネス

「◯◯レースを分析して」と頼むだけで、**観点別の並列web調査**を行い、**二層モデル**で
**勝率表・妙味・買い目（単勝/複勝・三連複/三連単）**を出力する Claude Code ハーネス。
レース確定後に予測精度を検証し、観点・重みを改善するループを回せる。

> ⚠️ 出力は予測であり的中・回収を保証しません。賭けは自己責任で。賭け金/Kelly は提示のみ、実ベットは人間判断で。

## 二層モデル

```
能力レイヤー: 発揮能力 = 潜在能力 × 適性 × 状態 − 割引 (+偶然)  → 各馬の「自分の勝率」
市場レイヤー: 妙味     = 自分の勝率 − 市場の勝率                 → 買うか / 買い目を決める
```

- **潜在能力**＝能力指数・近走内容・血統、**適性**＝コース/距離/馬場・展開/枠、
  **状態**＝調教・ローテ・当日気配、**割引**＝出遅れ/気性難/脚部不安など減点。
- 市場勝率はオッズの逆数を控除率で正規化（overround 除去）して算出。
- 妙味プラス（市場が過小評価）かつ期待値 EV が閾値超の馬を買い目候補に。

## 使い方

| コマンド | 役割 |
|----------|------|
| `/analyze-race <レース名 \| netkeiba URL \| 出走表貼り付け>` | 観点別並列調査 → 勝率表・妙味・買い目を出力 |
| `/review-prediction <レース名> <結果>` | 予測 vs 実結果を検証し、重みの改善案を提示 |

### 例

```
/analyze-race 2026 日本ダービーを分析して
/analyze-race https://race.netkeiba.com/race/shutuba.html?race_id=...
/review-prediction 20260531-tokyo-11 の結果でレビュー
```

出走表・オッズ・当日気配など崩れやすい核データは、web 取得が不確実なとき
URL 指定または貼り付けでの補強を依頼します。

## 仕組み（観点別ファンアウト）

`/analyze-race` は観点 A〜J（下表）ごとに調査サブエージェントを**並列起動**（内部で Workflow を使用）。
各エージェントが全出走馬についてその観点の好材料/懸念・寄与スコア・出典を返し、
メイン文脈が `scoring-model` に従って発揮能力→勝率→妙味→買い目を計算します。

| 因子 | 観点 |
|------|------|
| 潜在能力 | A 能力指数・時計 / B 近走内容・クラス通用度 / C 血統 |
| 適性 | D コース・距離・馬場適性 / E 展開・脚質×枠・ペース |
| 状態 | F 調教・厩舎 / G ローテ・馬体・間隔 / H 当日気配・パドック |
| 割引 | I リスク/割引要因 |
| 市場 | J オッズ・人気・市場勝率 |

頭数・情報量に応じて 6 / 8 / 10 観点を切り替えます。

`race-id` は `YYYYMMDD-開催-レース番号`（例: `20260531-tokyo-11`）。

## 改善ループ

1. `/analyze-race` で予測 → `predictions.jsonl` に追記。
2. レース確定後 `/review-prediction` で `results.jsonl` に結果記録。
3. 予測と結果を突合 → キャリブレーション（勝率ビン別的中率・Brier スコア・ROI）を算出。
4. 系統的に過大/過小評価の観点を診断 → `scoring-model.md` の重み・閾値の修正案を提示。
5. 承認で重みを更新（バージョンを上げる）→ これが 1 周。
